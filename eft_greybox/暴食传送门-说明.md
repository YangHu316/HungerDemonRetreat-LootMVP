# 🌀 暴食传送门 · 外卖叠叠乐 — Godot 白盒说明

> 严格遵循《策划文件/小游戏策划案/饿魔退散外卖侠-小游戏-暴食传送门.md》。
> 引擎：**Godot 4.7**。这是 **2D 灰盒/白盒**——重在"俄罗斯方块掉落 + 同稀有度相邻合成 + 溢出撕裂散落抢救"的玩法骨架，不含精细美术/音效。

## 怎么跑
1. 用 **Godot 4.7** 打开本项目（`Godot_Demo/eft-delivery/project.godot`）。
2. 按 **F5**（主场景为开发沙盒 `dev_launcher.tscn`）：**F2** 切换小游戏、**F1** 开发者模式（启用 `R` 测试键）。
   - 本场景已是**可实例化进大地图的 2D UI 组件**（`standalone` 默认 false，`begin()` 前不绘制）。接入方式 / 可替换美术槽（食物贴图·传送门贴图）/ 开发者模式 详见《接入大地图与开发者模式-说明》。

## 操作
| 操作 | 作用 |
|------|------|
| **←/→ 或 A/D** | 左右移动当前下落件 |
| **↑ 或 W** | 旋转 90° |
| **↓ 或 S** | 软降（按住加速下落） |
| **空格** | 硬降（直接落底锁定） |
| **鼠标左键**（撕裂后） | 抢救散落的食物（点中即入包，优先捡贵的） |
| **F**（撕裂后） | 弃货逃走，立即结束 |
| **R** | 重开一局 |

## 玩法流程（对应策划案）
1. **传送门掉落**：一局固定掉 **N=14** 件，**只掉 ⚪🟢🔵**（44/33/22%）。稀有度决定形状/占格：⚪1 / 🟢2 / 🔵3 / 🟣4 / 🟡5 格。此外会偶尔掉**变质堵块**（馊绿褐色、不可合成、只占地、打断同色簇）。
2. **整理堆叠**：移动/旋转/软降/硬降，把不规则形状塞进 **5×6** 背包。
3. **自动合成**：同稀有度**正交相邻成簇**即合成为高一级，**从顶部重新下落**（可连锁）。**⚪🟢🔵 需 2 连、🟣 需 3 连**（2蓝→紫、**3紫→金**，仅顶阶 3 连让金色稀有）。**🟣🟡 只能这样合成得到、不掉落；🟡 封顶**。
4. **胜利**：N 件全部落定且不溢出 → 全部食物**整齐入可搜刮容器**（显示清单与总价值）。
5. **撕裂（溢出）**：顶部塞不下 → **巨响 + 食物散落一地**（不入容器）→ 进入**限时抢救**（敌人 ETA 倒计时）：**点击捡高价值入包，或按 F 逃走**；倒计时结束未捡的丢失。

## 可调参数（选中根节点 `GluttonyPortalGame`，Inspector）
- `drop_count` 每局掉落数量 N（默认 14）—— 🔴 与背包尺寸共同决定胜率，**需手调**
- `fall_interval` 下落间隔（手感）
- `salvage_window` 撕裂后抢救窗口/敌人 ETA（默认 7s）—— 🔴 决定"紧张 vs 挫败"
- `garbage_rate` 变质堵块出现率（默认 0.18）/ `max_garbage` 每局上限（默认 5）—— 🔴 提难干扰物
- 合成所需连接数在脚本常量 `MERGE_NEED`（**2🔵→🟣、3🟣→🟡**；即 ⚪🟢🔵 2 连 · 🟣 3 连）—— **改难度核心改这里**（想让金色更难：把 🟣 提到 4 连，或把 🔵 改回 3 连）

## 灰盒取舍 / 已知偏差（待后续）
- **无沉降(settle gravity)**：合成移除两件后，上方已锁定件**不下沉**（保持原位、留空格）。连锁靠"升级件重新下落后再相邻"触发，而非塌缩。策划案把沉降列为 🔴 实现+手调项——正式版再加。
- **每稀有度仅 1 种 canonical 形状**（⚪■/🟢双/🔵L/🟣田/🟡P）：策划案的"形状池"未全做，先验证机制。
- **2D 占位**：纯色块 + 食材名文字；正式版按 §六 做 2.5D 面板 + 低模食材。
- **撕裂散落 = 面板内抢救**：散落物在屏幕上点击抢救（敌人为抽象 ETA 条）。策划案的"撒进 3D 世界、真饿魔扑来抢"属主循环集成，白盒先用倒计时模拟那份紧张。
- **下一件预览**未做（策划案标为可关的 🔴 手感项）。
- **数值/形状全为起调值**：掉落数量、占格、价值、抢救窗口都需在引擎里手调（项目约定：数值=人主导）。
