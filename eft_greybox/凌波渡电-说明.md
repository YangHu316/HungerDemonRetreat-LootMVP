# ⚡ 凌波渡电（跳一跳）· Godot 白盒复现 — 说明

> 严格遵循《策划文件/小游戏策划案/饿魔退散外卖侠-小游戏-凌波渡电.md》。
> 引擎：**Godot 4.7**（Forward+ / Jolt）。**灰盒/白盒**实现——重在"蓄力精准跳 + 落水弹回 + 拉闸开路"的玩法骨架与手感可读，不含精细贴图/音效。

## 怎么跑
1. 用 **Godot 4.7** 打开本项目（`Godot_Demo/eft-delivery/project.godot`）。
2. 直接按 **F5**（主场景为开发沙盒 `dev_launcher.tscn`）：**F2** 切换小游戏、**F1** 开发者模式（启用 `R` 等测试键）。
   - 本场景已是**可实例化进大地图的组件**（`standalone` 默认 false）。接入方式 / 可替换美术槽（落脚点·玩家·电闸·水面等 3D 模型）/ 开发者模式 详见《接入大地图与开发者模式-说明》。

## 操作
| 操作 | 作用 |
|------|------|
| **长按 / 松开 [空格]** | 蓄力 → 起跳。按住越久蓄力越满、跳得越远，松手沿抛物线飞出（**无落点提示，靠手感判断**；落地停在真实落点、不吸附格中心） |
| **[E]** | 抵达对岸终点后拉电闸（约 1.2s）→ 全池断电 |
| **[R]** | 重置回合 |

## 玩法流程（与策划案 §一/§二 对应）
1. **通电水池拦路**：水面蓝白电光脉动 = "带电、碰了完蛋"。一串落脚点（间距不等）跨越水池。
2. **蓄力跳**：长按蓄力条上涨→对应跳距，松开起跳（**无落点可视化，靠手感判断**；落地不吸附、停在真实落点，下一跳从真实位置起跳）。蓄力不足/过头 → **落水**。
3. **落水触电**：电弧闪白 + 震屏 + 角色弹飞 → **弹回起点 + 损失时间**（`fall_reset_delay`，不致死可重试）。
4. **拉电闸开路**：到对岸按 E → 闸刀拉下 → **全池永久断电**（水面转安全暗色、电光熄灭、闸刀变绿）。
5. **后续轻松通过**：断电后自动出现 **2 名队友**直接趟水走过——演出"首名紧张 vs 后续轻松"的反差（设计灵魂）。

## 可调参数（选中根节点 `ElectricCrossingGame`，Inspector 调）
- `charge_time` 蓄满秒数、`jump_min_dist`/`jump_max_dist` 跳距区间、`jump_arc_height` 弧高、`jump_air_time` 腾空时间 —— **🔴 手感核心，需在引擎里手调**
- `fall_reset_delay` 落水损时（2~4s）、`switch_pull_time` 拉闸时长（1~1.5s）
- `landing_bonus` 落点判定宽容（=落脚点半宽 + 此值；调难度/手感）
- `follower_speed` 后续趟水移速、`sound_radius_splash` 声音半径接口

## 灰盒取舍 / 已知偏差（待美术/后续）
- **落脚点/水/闸刀/角色**均为 box/capsule 占位低模；蓄力/腾空/落地姿态用"压低缩放"近似，正式版需纸片角色姿态帧（§六 标 🔴）。
- **镜头不跟随抛物线飞行**（§五 列为加分项）：当前为固定斜俯视聚焦全池，已足够读跳跃；跟随镜头待手感阶段再调。
- **声音/震波**未发实声——`_emit_noise` 是接《感知与寻人》的接口位（落水反复 15m 巨响=持续招饿魔，那套压力在接系统后体现）。
- **三人合作**：本场景单人测试；后续玩家用自动 follower 模拟，正式版为真实队友在世界空间趟水。
- 蓄力为"长按时长"模型（非来回摆动的力度条）——若想要更强的"赌一把"节奏可改为往复力度条，属 🔴 手感取舍。
