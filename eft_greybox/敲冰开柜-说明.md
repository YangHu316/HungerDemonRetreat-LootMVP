# 🧊 敲冰开柜 · Godot 白盒复现 — 说明

> 严格遵循《策划文件/小游戏策划案/饿魔退散外卖侠-小游戏-敲冰开柜.md》。
> 引擎：**Godot 4.7**（Forward+ / Jolt 3D 物理）。这是**灰盒/白盒**实现——重在玩法与"渐进碎裂爽感"，不含精细贴图/音效（策划案 §十三 明确白盒无需这些）。

## 怎么跑
1. 用 **Godot 4.7** 打开本项目（`Godot_Demo/eft-delivery/project.godot`）。
2. 直接按 **F5**（主场景为开发沙盒 `dev_launcher.tscn`）：**F2** 切换小游戏、**F1** 开发者模式（启用 `1/2/3`/`R` 等测试键）。
   - 本场景已是**可直接实例化进大地图的组件**（`standalone` 默认 false）。接入方式 / 可替换美术槽（冰柜·食物·碎片模型）/ 开发者模式 详见《接入大地图与开发者模式-说明》。

## 操作
| 操作 | 作用 |
|------|------|
| **鼠标左键** | 砸冰（对准玻璃连续点击；**破碎前可随时改敲别格**，各格各自累计） |
| **鼠标右键** | 擦玻璃（每柜 1 次，看清 1 格内容） |
| **1 / 2 / 3** | 冰柜大小 1×1 / 2×2 / 3×3 并重置（每格随机稀有度，体验不同点击数 4~20） |
| **R** | 重置当前回合 |

## 内容物图片 + 玻璃碎片（最新）
- **内容物用图片**：每件食物用 `assets/food/<名称>.jpg` 的图片显示（替代圆球），自动缩放容纳进格子；名称颜色不变；变质时图片变暗。16 张图片已按菜名生成并下载。**缺图会自动回退为纯色占位——想换更合适的图，直接替换 `assets/food/` 里的同名 .jpg 即可。**
- **玻璃碎片 ×3 且停留**：柜门破碎时碎片 16 → **48 片**，落地后**停留在场上**（不再 4 秒自动消失；仅 R 重置 / 换柜时清空）。
- 实现：图片用 `Image.load` 运行时加载原始文件——编辑器内 F5 运行即可见（无需手动导入）。

> 注：现有 16 张是按菜名自动生成的示意图，非人工逐张精挑。要更精准可自行替换同名文件。

## 最新一轮新增（2026-06-22）
- **稀有度生成概率**：白 40% / 绿 30% / 蓝 20% / 紫 9% / 金 1%（`rarity_weights` 可调）。实测 1 万次 ≈ [3974, 2980, 2024, 909, 113]。
- **毛玻璃 + 擦玻璃**：初始柜门毛玻璃**看不清内容**；每柜给 **1 次擦玻璃**（右键，只能擦 1 格）→ 擦开的格能看清内容与名称；其余格保持盲盒，靠赌。
- **内容物名称**：每件食物有名字（命名库），**悬浮在内容上方、颜色＝品质色**。砸开后**所有格名称揭示**——让你看清"选了什么、舍弃了什么"（如舍弃了金色佛跳墙）。
- **中文字体修复**：Godot 默认字体无中文，已用系统字体（微软雅黑 / SimHei…）渲染所有中文，避免方块。
- **命名库**：白[米饭/馒头/面包/地瓜/玉米/挂面] · 绿[包子/饺子/炒饭/蘑菇汤] · 蓝[汉堡/盖饭/水果蛋糕] · 紫[黯然销魂饭/烧鹅] · 金[佛跳墙]。

## 早前新增功能
- **冰柜大小**：1×1 / 2×2 / 3×3（键 1/2/3 切换），每格一件随机稀有度食物、透出光色。
- **只能开一格**：首次点击锁定该格，之后只能砸这一格（盲盒抉择＝选了就放弃别的）。
- **其余变质失效**：被选格砸开后，其余格食物全部 **发黑 + 冒绿色烟雾**（选择代价）。
- **Bug 修复**：玻璃碎裂时 **裂纹随之消失**（不再悬浮空中）。

> ⚠️ **设计一致性提醒**：这条"多格单选 + 未选格变质"是《敲冰开柜》策划案**最初有、后被改掉**的设定（现行案写的是"取消多冰格单选盲盒 + 未选格报废，改为单件食物连点砸碎"）。本实现按你本次要求把它**加回来**了。要不要我同步把策划案更新为"**多格盲盒选择 + 连点砸碎**"的合并版，保持文档与实现一致？

## 可调参数（选中场景根节点 `IceCabinetGame`，在 Inspector 调）
- `rarity`：默认稀有度
- `click_counts`：各稀有度点击数（⚪4/🟢6/🔵10/🟣15/🟡20）
- `crack_stages`：裂纹分级数（默认 4）
- `refreeze_delay`：中断回冻时长（默认 5s）
- `show_rarity_color`：稀有度光色透显开关（关=盲砸惊喜）
- `爽感 Juice` 组：每击/爆碎震感、震感随进度增长、顿帧时长（§五 标注的"人手调"项）

## 灰盒取舍 / 与策划案的已知偏差（待美术/后续）
- **裂纹**用"世界线段投影到 2D 叠层"绘制（清晰可读），非玻璃表面 shader——白盒做法，便于看进度。
- **镜头**为固定斜俯视聚焦（§五 的"进入 0.4s lerp"已做；但不含"在 3D 世界里走近柜子"的接入，那属主循环集成）。
- **音效/声波环**未做（§七 声音半径是接感知系统的接口，白盒不需要实声）。`_register_hit` 里已留每击位置，接声音系统时在此发声即可。
- **三人合作同砸**：本场景为单人测试；计数走 `_register_hit()`，多人时让各客户端命中都调用它即可共享计数（§八/§十）。
