# 《饿魔退散！外卖侠》饕餮双环转盘供宴改版草案 v3

> 本文档为“饕餮转盘供宴”的玩法改版草案。  
> v2：保留旋转圆盘/投掷/投喂核心乐趣，把场地从“锁死扇形区”改为“双环自由移动 + 任意位置投喂”。  
> **v3 难度模型更新（已拍板，见 §〇）**：执行案 v1 已锁定“固定三人、不做 1/2 人适配”，故本文档原“按人数适配难度”（§六、§九、§十）**降级为旋钮取值参考**；难度改为**按遭遇时间递增**，胜负改为**条件式奖励**。两项与《可配置参数总表 v1》§13 同源。

## 〇、难度模型更新（v3 新增 · 已拍板 · 统领全文）

> 下文 §六/§九/§十按“单/双/三人”描述的难度适配，**已被“固定三人 + 时间难度档”取代**；那些参数表保留为**旋钮取值范围参考**。本节与它们冲突时**以本节为准**，并与《可配置参数总表 v1》§13 同源。

### 0.1 难度改按“遭遇时间”递增

固定三人合作，难度不再随人数变，而是**随单局时间推移而递增**：饿魔越来越强，体现在两条线——

1. **主地图追人速度**：饕餮巡逻/搜寻移速、锁定激进度随时间提升，越后期越难甩开（§0.3）。
2. **供宴难度**：被抓后进入的转盘，按“被吸入那一刻的单局时间”**快照**配置难度档，越晚被抓越刁钻（§0.2）。

**为什么这样改**：原方案担心“双环精心做了却因玩家总能躲掉而很少被看到”。时间递增正好解决——前期同向慢转的温和供宴是软教学；后期饕餮变快、更容易真被抓，此时双环的完整复杂形态（反向 + 挡板 + 快转）才登场，而你正背着深区高价值易腐食物，被抓最肉痛、奇观也最值。

### 0.2 供宴时间难度档（被吸入瞬间快照，进入后该局维持）

| 档位 | 单局时间 | 内外圈 | 转速 | 挡板 | 中心 | 需求 | 食物色数 | 怒气增速 |
|------|---------|--------|------|------|------|------|---------|---------|
| **T1 温和** | 0–5min | **同向** | 慢 ~0.12 | 0 | 可通行 | 单目标·时限宽 | 5 | 慢 |
| **T2 升温** | 5–10min | **反向** | 中 ~0.18 | +1~2 | 可通行 | 双目标·双色组合 | 6（+红） | 标准 |
| **T3 刁钻** | 10–15min | 反向 | 快 ~0.22 | +3~4 | **不可通行** | 三目标·唯一目标接力·刁钻配色 | 7（+粉） | 快 |
| **狂暴/迷失线** | **15:00 硬上限** | 反向 | 最快 | 4 | 不可通行 | 最刁钻 | 7 | 最快 |

> 这些旋钮 demo 已全部实现可实时调（内外同向/反向开关、转速滑条、挡板 0~4、中心可通行开关、红/粉食物开关）——**demo 即本难度递增的现成调参台**。需求复杂度走 demo 事件脚本的单→双→三目标渐进。
>
> **护栏**：难度靠“速度 + 刁钻”抬，不靠拉长时长。一次惊动总成本随档略升（~15s → ~30s）但始终 ＜ 转盘 90s，“躲优先”不破。

### 0.3 主地图追人速度递增（细化）

| 档位 | 巡逻移速 | 搜寻移速 | 放弃倒计时 | 视线锁定阈值 | 体感 |
|------|---------|---------|-----------|-------------|------|
| T1 | 基准×0.85 | 基准 | 短 ~8s | ~1.8s | 容易甩开，多数遭遇降噪走开即可 |
| T2 | 基准 | 基准×1.1 | 标准 ~10s | ~1.5s | 要认真断视线/钻躲点 |
| T3 | 基准×1.15 | 基准×1.25 | 长 ~12s | ~1.2s | 很难空手甩开，常需臭包/引怪/躲点 |
| 狂暴/迷失线 | 最快 | 最快 | 最长 | 最短 | 最后冲刺；15:00 未撤=迷失 |

> 倍率为 v1 起调值，与《饿魔感知与寻人逻辑》§二计时器表对齐后灰盒调。

### 0.4 胜负结算：条件式奖励（取代原五档）

| 结果 | 条件 | 效果 |
|------|------|------|
| 🏆 完胜 | 饱食满 **且 全程零怒气** | 吐 1 份额外奖励食物 |
| ✅ 达标 | 饱食满 **但 曾积累怒气** | 零损失，无奖励 |
| ⚠️ 失败 | 饱食未满 | 随机丢食物（≥70% 丢 1 / <70% 丢 2 / 怒气爆丢 3） |

> 奖励只给“零瑕疵”，且随难度档越来越难拿 → 被抓的现实最优几乎都是“白花 90s”的零损失 → 故意被抓永不划算，“躲优先”成立。详见《可配置参数总表 v1》§13.3。

## 一、改版核心方向

### 1.1 关键变化

本版不再采用“玩家被锁在固定扇形区，只有经过巨口的区块才能投喂”的规则，而改为：

> **玩家可以在内圈或外圈的可行走区域内自由移动，但会受到所在圆环旋转带动，并被内外圈挡板阻拦。投喂不再限定在外圈窗口，只要玩家能把正确食物投进饕餮口中，就算成功投喂。**

### 1.2 新版核心体验

| 体验 | 说明 |
|------|------|
| **移动中的场地** | 玩家脚下的内圈/外圈会持续旋转，站在上面会被圆环带着移动 |
| **挡板制造路线限制** | 内圈和外圈都有4个分隔区，挡板会阻挡玩家直接穿越，玩家需要绕行或等待相对位置变化 |
| **投掷成为核心操作** | 食物不一定要送到某个窗口，只要投进饕餮口中即可 |
| **人数越多越好配合** | 多人可以分头找食物、接力传递、补位投喂，因此天然更容易 |
| **人数越少越降难度** | 单人/双人通过降低转速、增加目标食物掉落数量、放宽投喂判定来保持可玩 |

---

## 二、新玩法一句话规则

> 饕餮巨口固定在场地外侧，内圈和外圈各有4个被挡板分隔的旋转区块；内圈顺时针旋转，外圈逆时针旋转。玩家可以在所在圆环区域中自由移动，但会被旋转带动并被挡板阻挡。食物随机散落在场地内，玩家需要寻找饕餮要求的目标食物，通过移动、投掷、接取、传递，把正确食物投进饕餮口中。

## 三、场地结构

### 3.1 双环旋转餐桌

```text
                         饕餮巨口
                            ↓
                       【投喂目标】

              ┌───────────────────────┐
              │       外圈 O1          │
        ┌─────┼─────┬─────┬─────┼─────┐
        │ O4  │     │     │     │ O2  │   外圈逆时针旋转
        │     │ I4  │ I1  │ I2  │     │
        │     │     │内圈 │     │     │   内圈顺时针旋转
        │ O3  │ I3  │     │     │     │
        └─────┴─────┴─────┴─────┴─────┘

        内圈/外圈都有挡板，玩家无法无视挡板直线穿越
```

### 3.2 场地元素

| 元素 | 数量/规则 | 功能 |
|------|-----------|------|
| 内圈 | 4个分隔区，顺时针旋转 | 生成食物、提供近距离绕行路线 |
| 外圈 | 4个分隔区，逆时针旋转 | 生成食物、靠近饕餮投喂距离更近 |
| 挡板 | 内外圈各4组 | 阻挡玩家移动路线，制造绕行与等待 |
| 饕餮巨口 | 固定在场地外侧 | 唯一投喂目标，可从任意位置投掷命中 |
| 食物散落点 | 内外圈均有 | 随需求刷新目标食物和干扰食物 |
| 掉落边界 | 圆盘外侧/裂缝 | 食物投出场地后消失并增加怒气 |

### 3.3 内外圈反向旋转的意义

| 设计 | 产生的体验 |
|------|-----------|
| 内圈顺时针 | 玩家站在内圈时会被带向一个方向，需要持续修正路线 |
| 外圈逆时针 | 外圈玩家与内圈玩家的相对位置不断变化，制造传球窗口 |
| 双圈错位 | 某一刻离队友很远，下一刻可能刚好贴近，适合接力投掷 |
| 挡板阻拦 | 玩家不能只走直线，必须判断绕行、等待、投掷哪个更快 |

## 四、玩家移动规则

### 4.1 自由行走，但受旋转影响

玩家不是按键切换分区，而是在当前可行走区域内自由移动。

| 行为 | 规则 |
|------|------|
| 区域内移动 | 玩家可以在当前内圈/外圈的可通行地面自由移动 |
| 圆环带动 | 玩家会继承脚下圆环的旋转位移，站着不动也会被带走 |
| 主动移动 | 玩家输入移动方向时，是在“自身移动 + 圆环带动”叠加状态下移动 |
| 挡板阻挡 | 内外圈挡板会阻止玩家直线穿越，需要绕过挡板或等待转动带来通路 |
| 内外圈关系 | 内外圈之间可设计若干连接缺口/过道，玩家可以自然走过去，但会受到挡板和旋转影响 |

### 4.2 不再使用“换环按键”

本版删除“经过换环窗口按E切换内外圈”的规则。

改为：

> **玩家通过真实移动进入内圈或外圈的连接通路。是否能顺利过去，取决于当前挡板位置、圆环旋转方向和玩家移动判断。**

这样体验更直觉：玩家不是被系统传送，而是在一张会转动的危险餐桌上跑位。

### 4.3 移动难点

玩家需要同时处理三件事：

1. 自己要去哪里拿食物。
2. 脚下圆环正在把自己带向哪里。
3. 挡板是否会挡住最短路线。

这让移动本身成为轻度派对混乱来源。

## 五、投喂规则

### 5.1 任意位置可投喂

本版投喂不再限定外圈窗口。

只要玩家能把正确食物投进饕餮口中，即可算作投喂成功。

| 投喂方式 | 说明 |
|---------|------|
| 近距离投喂 | 玩家靠近饕餮所在方向，直接抛投进巨口，命中率高 |
| 中距离投喂 | 需要判断抛物线和圆盘移动，难度中等 |
| 远距离投喂 | 可以尝试，但抛物线更长、落点更难、失败风险更高 |
| 队友接力投喂 | 一名玩家把食物传给更靠近饕餮的队友，由队友完成最终投喂 |

### 5.2 为什么允许任意位置投喂

| 原投喂窗口限制 | 新任意投喂 |
|---------------|-----------|
| 强制玩家等外圈到位 | 玩家可以主动寻找角度和距离 |
| 更像固定节奏机关 | 更像混乱派对投掷玩法 |
| 单人容易被窗口卡死 | 单人可通过降低难度和多目标食物主动投喂 |
| 多人配合点固定 | 多人可以自由分工：找菜、传球、靠近投喂、远投补救 |

### 5.3 投喂判定

| 情况 | 结果 |
|------|------|
| 正确食物命中饕餮口 | 饱食槽增加 |
| 错误食物命中饕餮口 | 怒气增加 |
| 食物砸到饕餮脸但未入口 | 食物弹飞，少量怒气 |
| 食物掉出圆盘 | 食物消失，怒气增加 |
| 食物落在场地内 | 可继续拾取/传递 |

### 5.4 抛物线辅助

投喂和传递都使用同一套抛物线系统。

| 辅助 | 说明 |
|------|------|
| 抛物线轨迹 | 显示食物飞行路径 |
| 预测落点 | 显示食物落地时对应的旋转后位置 |
| 饕餮口命中提示 | 抛物线终点覆盖饕餮口时，巨口边缘高亮 |
| 队友软锁定 | 准星靠近队友时轻微吸附，方便传递 |
| 出界警告 | 预测落点会掉出场地时显示红色 |

## 六、食物生成规则

### 6.1 食物随机散落

食物依旧随机散落在内圈和外圈各处。

| 类型 | 说明 |
|------|------|
| 目标食物 | 当前饕餮需求所需食物 |
| 干扰食物 | 非当前需求食物，制造识别和选择压力 |
| 高价值食物 | 紫色需求或最后一口需求出现 |

### 6.2 根据人数调整目标食物数量

新版主基调为：

> **人越多越容易玩，但人数减少时系统会稍微降低难度，让单人和双人仍然成立。**

| 人数 | 目标食物掉落 | 圈速 | 投喂判定 | 合作要求 |
|------|-------------|------|---------|----------|
| 单人 | 目标食物掉落多个，分布更接近玩家 | 慢 | 判定较宽 | 玩家自己找、自己投，偏操作和路线规划 |
| 双人 | 目标食物掉落2个左右，分布在两人可接力的位置 | 中 | 标准偏宽 | 鼓励传递，但不强制唯一解 |
| 三人 | 目标食物可能只有1个，且位置更远/更分散 | 快 | 标准 | 必须传递唯一正确目标食物，强调配合 |

### 6.3 三人模式的核心张力

三人时，系统可以设计：

```text
饕餮需求：蓝色×1
场上只掉落1个蓝色目标食物
目标食物出现在离饕餮较远的位置
三名玩家必须判断：谁去拿、谁接力、谁最终投喂
```

这会自然产生喊话：

- “蓝的在内圈左边！”
- “我被挡板卡住了，丢给你！”
- “我离嘴近，我来喂！”
- “别直接投，角度不够！”

### 6.4 单人模式的核心张力

单人时，不强求传递，而是降低难度：

- 目标食物更多
- 圈速更慢
- 饕餮口判定更宽
- 怒气增长更慢

单人玩法重点变成：

> 找到目标食物 → 对抗旋转移动 → 找到合适角度 → 投进饕餮口。

## 七、饕餮需求设计

### 7.1 需求类型

需求规则保持简单，只使用颜色+数量。

| 需求类型 | 示例 | 单人 | 双人 | 三人 |
|---------|------|------|------|------|
| 单色单件 | 🟢×1 | 常见 | 常见 | 教学/过渡 |
| 单色双件 | 🔵×2 | 少量 | 常见 | 常见 |
| 双色组合 | 🟢+🔵 | 少量 | 常见 | 常见 |
| 高价值一口 | 🟣×1 | 最后一口 | 最后一口 | 最后一口 |

### 7.2 不加入复杂需求

暂不加入：

- 形状需求
- 顺序需求
- 禁忌食物
- 记忆需求

原因：玩家已经需要处理旋转、挡板、投掷、接取，需求规则必须保持低负担。

## 八、投掷与接取机制

### 8.1 传递价值

虽然任意位置都能投喂，但传递仍然重要。

| 情况 | 为什么需要传递 |
|------|---------------|
| 拿到食物的人离饕餮太远 | 直接远投命中率低，传给近处队友更稳 |
| 玩家被挡板阻拦 | 食物可以飞过部分阻挡区域，节省绕路时间 |
| 圈速较快 | 食物持有者可能被旋转带离好角度，需要传给别人 |
| 三人唯一目标食物 | 一旦拿到正确食物，团队要保护并接力它 |

### 8.2 接取规则

| 情况 | 结果 |
|------|------|
| 食物投到队友附近 | 队友自动接住或一键拾取 |
| 食物落在可行走区域 | 停在地面，可被拾取 |
| 食物碰到挡板 | 弹回或沿挡板滑落 |
| 食物掉出圆盘 | 消失并增加怒气 |
| 食物落入饕餮口 | 若符合需求则成功投喂 |

## 九、人数难度适配

> ⚠️ **已被 §〇 取代**：执行案 v1 锁定“固定三人”，本节"按人数适配"不再作为难度模型，**仅保留为各旋钮的取值范围参考**（转速/挡板/判定宽度等）。当前难度按**遭遇时间档 T1/T2/T3**（§0.2），不按人数。

### 9.1 总原则

> 人越多，玩法越容易；人越少，系统降低难度但不完全抹平差距。

也就是说：

- 三人是最热闹、最容易形成接力的版本。
- 双人仍可稳定完成，但需要更谨慎判断。
- 单人可以完成，但更偏操作挑战。

### 9.2 难度参数表

| 参数 | 单人 | 双人 | 三人 |
|------|------|------|------|
| 圆盘转速 | 慢 | 中 | 快 |
| 目标食物数量 | 3个左右 | 2个左右 | 1个为主 |
| 目标食物位置 | 更靠近玩家/更易到达 | 分布在两人可接力范围 | 更远、更分散 |
| 饕餮口判定 | 宽 | 标准偏宽 | 标准 |
| 怒气增长 | 慢 | 标准 | 标准或略快 |
| 需求复杂度 | 单件为主 | 单件+少量双件 | 单件、双件、双色组合 |
| 掉落出界惩罚 | 低 | 标准 | 标准 |

### 9.3 单人体验

单人核心是：

> “我能不能在旋转和挡板干扰下，自己找到目标食物并投进饕餮嘴里。”

不强调传递，强调路线、角度和操作。

### 9.4 双人体验

双人核心是：

> “我们能不能判断谁更适合拿菜，谁更适合投喂。”

传递开始变重要，但目标食物不应唯一到让失败过于挫败。

### 9.5 三人体验

三人核心是：

> “唯一正确食物出现后，三个人能不能快速完成接力，把它送进饕餮口中。”

三人时应更强调唯一目标食物、快速传递、补位和喊话。

## 十一、相比原版的优劣

### 11.1 优点

| 优点 | 说明 |
|------|------|
| 支持单人/双人/三人 | 不再锁死固定三人结构 |
| 操作更直觉 | 玩家自由走动，不通过按键切区 |
| 投喂更自由 | 只要能投进饕餮口中，任意位置都可投喂 |
| 人数越多越有优势 | 多人自然更容易找菜、接力、补位 |
| 派对感更强 | 旋转、挡板、投掷失误会制造混乱和笑点 |

### 11.2 风险

| 风险 | 说明 | 缓解 |
|------|------|------|
| 规则复杂度上升 | 双环反向旋转+挡板+自由投掷信息量较大 | 需求规则保持简单，只用颜色+数量 |
| 程序量上升 | 需要处理旋转平台、挡板碰撞、移动叠加 | 先做灰盒验证 |
| 单人可能过忙 | 单人要找菜、走位、投喂 | 降低速度、增加目标食物、放宽判定 |
| 投掷挫败 | 目标和玩家都在移动 | 抛物线预测落点和饕餮口命中提示必须清晰 |

## 十三、当前建议

这个版本比上一版更符合你的新目标：

- 不再被固定三人结构限制
- 单人和双人可以成立
- 多人依然更有优势
- 投喂更自由，更像派对投掷玩法

但它对程序验证要求更高，建议先单独做一个灰盒测试场景验证：

1. 玩家站在旋转内圈/外圈上时移动是否舒服。
2. 挡板阻拦是否会制造有趣路线，而不是卡玩家。
3. 任意位置投掷进饕餮口是否清楚、可控。
4. 三人唯一目标食物接力是否真的好笑、紧张。
5. 单人降低难度后是否依然有挑战。

## 十四、总结

新版双环转盘供宴的核心不是“等某个区块到嘴边”，而是：

> **在两个反向旋转、带挡板阻拦的餐桌环上，寻找正确食物，通过移动、接力和抛物线投掷，把食物直接投进饕餮巨口。**

它的体验基调是：

- 单人：低速、多目标、路线与投掷挑战
- 双人：中速、适量目标、传球配合
- 三人：高速、目标稀少、唯一正确食物接力

一句话概括：

> **这是一个“反向旋转双环餐桌上的合作投喂躲避球”，人数越多越容易形成配合，人数越少则通过降低转速和增加目标食物来保证可玩。**
