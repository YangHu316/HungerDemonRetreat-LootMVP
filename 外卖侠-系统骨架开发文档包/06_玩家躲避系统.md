# 《饿魔退散！外卖侠》玩家躲避系统

> 系列文档之一。回答你提出的三个问题：**躲在哪、怎么躲、躲多久**。
>
> 配套阅读：《饿魔感知与寻人逻辑》（本系统是它的另一半）。**核心原则**：遭遇饿魔时，**躲是最优解**，因为它最省时；转盘是"没躲掉"的代价。

---

## 〇、躲避的三个层级（从轻到重）

> "躲"不止"钻柜子"。最省时的躲，往往是**安静走开**。三层按时间成本递增，玩家应优先用低层级。

```text
威胁升级 ─────────────────────────────────────────▶
层级1 降噪行动   →   层级2 断视线   →   层级3 钻躲藏点
(最省时，首选)        (有视线时)         (近身/搜寻态时)
```

| 层级 | 做什么 | 适用情形 | 时间成本 |
|------|--------|---------|---------|
| **① 降噪行动** | 停止高噪动作，切**静步**，离开声源点 | 饿魔刚被惊动、还在"调查"声源点 | 最低，几秒 |
| **② 断视线** | 用墙 / 建筑 / 容器挡住饿魔视野锥 | 饿魔有了你的视线 | 低 |
| **③ 钻躲藏点** | 进入躲藏点，静止等它放弃 | 饿魔已贴近 / 进入"搜寻"态、跑不掉 | 中（约 15~25s） |

**层级 ①** 之所以是首选：饿魔被惊动后扑向**声源点**而非你（见《感知逻辑》），你安静走开，它会扑空并在 15~25s 内放弃。绝大多数遭遇用层级①+②就能化解，根本用不到钻柜子。

## 一、躲在哪：躲藏点（沿用 v7.3 §8.2）

| 躲藏点 | 容纳人数 | 被发现概率(v1) | 特点 |
|--------|---------|---------------|------|
| 🗄️ 储物柜 | 1 人 | 10% | 经典位，最安全 |
| 🛏️ 床底 | 1 人 | 20% | 能看见饿魔脚步，紧张 |
| 📦 纸箱堆 | **2 人** | 20% | 两人挤一起 = 喜剧名场面 |
| 🪑 桌底 | 1 人 | 35% | 视野好但遮蔽差，最危险 |

> **被发现概率**：仅在饿魔处于"搜寻"态且**在极近处搜索到该躲藏点**时判定一次。平时躲着不会被无故发现。
>
> 躲藏点的**布置**见《地图尺寸与路线规划》：沿推进路线分布，商业区/深区更密集（那里饿魔更活跃）。
>
> **实现分工（v7.5）**：程序上只分 **容量 1 / 容量 2** 两种功能型；上表四种只是**美术模型 + 被发现概率数值**的差异，共用同一套躲藏逻辑——美术可继续加多样模型，不增程序复杂度。

## 二、怎么躲：交互与规则

| 规则 | 内容 |
|------|------|
| 进入 | 靠近躲藏点按 **E** 进入；角色静止 |
| 声音 | 躲藏中玩家发声半径 ≈ **0**（静止不动） |
| 视线 | 躲藏点**遮断**饿魔视线，它无法直接锁定你 |
| **不做屏息玩法** | **没有屏息条、没有 QTE**。要不要出来完全由玩家判断（沿用 v7.3） |
| 退出 | 随时按 E 离开 |
| 纸箱堆 | 可容纳 2 人，制造"两人挤一团"的喜剧 |
| 被发现 | 若判定被发现 → 升级为**锁定** → 3 秒吸入 → 转盘（代价） |

**紧张感来源（不是屏息，而是）**：
- **变质压迫**：你躲着不动时，背包里的食物在持续变质——躲着就是在亏钱/亏新鲜度。
- **决策焦虑**：饿魔在外面徘徊，"现在出来安全吗？"由你自己赌。
- **躲藏点选择**：身边只有桌底（35% 被发现）还是储物柜（10%）？

## 三、躲多久：等饿魔放弃

> "躲多久"由《感知逻辑》的计时器决定，这里给出玩家视角的换算。

| 情形 | 玩家需要躲多久 |
|------|---------------|
| 饿魔在"调查"态扑向声源点 | 你安静/躲约 **8~12s**，它搜完声源点就放弃 |
| 饿魔升级到"搜寻"态 | 约 **15~25s**，无新线索后回巡逻 |
| 你在躲藏中又发声/被瞥见 | 计时**重置**，会拖更久 → 所以躲着别乱动 |

**关键数值结论**：一次完整躲避 ≈ **15~25s**，**远小于**被卷入转盘的 **90s**。所以"躲"在时间账上永远优于"被抓"——这正是"躲优先"的设计落点。

## 四、合作躲避

| 合作行为 | 说明 |
|---------|------|
| 挤纸箱堆 | 两名玩家共用一个纸箱堆，喜剧 + 互相确认"它走了没" |
| 分头躲 | 三人分散躲不同点，降低被一锅端的概率 |
| 引怪掩护 | 一人用奔跑/投掷/制造噪音把饿魔引开（见《感知逻辑》合作维度），另两人趁机躲好或继续搜 |
| 臭包开窗口 | 丢臭包把饿魔从某区域推开 10s，给躲/撤争取窗口 |
| 语音报点 | "它在床那边""往左走了，可以出来了"——躲藏中最自然的沟通 |

## 五、设计护栏：别让"躲"变成无脑安全

> 这是必须正视的一点。既然躲又省时又安全，如果没有约束，玩家会"无脑躲"，核心取舍就被架空。约束来自**全局压力**：

| 压力源 | 如何让"躲太多"也是一种亏 |
|--------|------------------------|
| 全局倒计时（15min） | 躲掉的时间不会回来；倒计时归零后饿魔密度与攻击性大幅提升（v7.3 §10.1） |
| 食物变质 | 躲着不动 = 背包食物持续降档 = 报酬缩水 |
| 没搜够 | 时间花在躲上，就没花在搜刮/解障上 = 战利品更少 |

**结论**：躲是"遇敌时的最优解"，但**"少遇敌、少需要躲"才是更高水平的玩法**——靠声音管理、路线规划、临时道具，从一开始就别招饿魔。这样"贪 vs 稳"的取舍依然成立：躲得起一次两次，躲不起一整局。

## 六、灰盒验证清单

1. 玩家是否自然地先用"降噪走开"，而不是一上来就钻柜子？
2. 躲一次的时间成本是否被玩家感知为"比打转盘划算"？
3. 没有屏息条，玩家是否仍然紧张（靠变质 + "现在出来安全吗"）？
4. 纸箱堆挤两人是否真的好笑？
5. 全局压力是否足以阻止"无脑躲"——玩家会不会因为"躲太久食物凉了"而后悔？
