## 一、系统总览与单局数据流

### 1.1 系统关系图

```text
            ┌──────────────── 主界面（局外） ────────────────┐
            │  订单接取系统(§十)  ◀──读/写──▶  仓库系统(§九)   │
            └───────────────────────┬────────────────────────┘
                                    │ 进入危险区（带着已接订单）
                                    ▼
   ┌───────────────────────── 危险区（局内）─────────────────────────┐
   │  时间系统(§三) ── 驱动 ──▶ 变质 / 15min硬上限 / 各类时长           │
   │       │                                                          │
   │  动作系统(§四) ── 发声 ──▶ 声音系统(§五) ── 惊动 ──▶ 怪物寻人      │
   │       │                                      │ 锁定→遭遇          │
   │  躲藏系统(§六) ◀── 让怪物放弃 ───────────────┘                   │
   │       │                                                          │
   │  搜刮(§七) ──产出──▶ 背包(§八) ── 食物随时间变质(§三)             │
   │       │  └── 转场系统(§十一)─[占位]─ 小游戏/双环（§二）           │
   │       ▼                                                          │
   │  撤离系统(§十二)：撤离成功→结算 / **15:00未撤=迷失(失全部背包)**    │
   └────────────────────────────────┬────────────────────────────────┘
                                    ▼
                    结算系统(§十三)：新鲜度 × 订单完成度 × 数量
                                    │ 报酬 / 剩余物资（迷失则无）
                                    ▼
                              回主界面 → 入仓库(§九)
```

### 1.2 单局数据流（一句话）

> **接单**（订单池抽取，决定地图保证生成什么）→ **进局**（动作发声、声音惊动怪物、躲藏让其放弃、时间驱动变质与 15min 硬上限）→ **搜刮**（容器产出食物 / 钥匙 / 占位战利品入背包）→ **场景互动**（占位按钮代替小游戏）→ **撤离**（成功入结算 / **15:00 未撤离 = 迷失，失去全部背包物资**）→ **结算**（新鲜度 × 订单完成度 × 数量）→ **入仓库**。

## 二、占位约定（本骨架最重要的一节）

> 所有"小游戏 / 派对游戏"在骨架版里都不实现玩法，只用**统一占位交互**替代。真小游戏日后实现同一接口即可无缝替换。

### 2.1 统一交互接口（建议）

```text
interface IInteraction {
    SoundEmit  onTrigger;     // 触发时发出的声音半径(接§五)
    float      stubDelay;     // 占位用：按下后多久出结果(默认0~1s)
    Result     Run();         // 骨架版=出占位结果；正式版=跑真小游戏后返回同型Result
}
Result = { 成功?:bool, 产出物:List<Item>, 代价:{丢失物?, 时间损失?}, 触发遭遇?:bool }
```

- **骨架版**：`Run()` 不进任何小游戏界面，直接（或 stubDelay 后）返回**占位 Result**。
- **正式版**：`Run()` 走《小游戏策划案》真玩法，返回同型 `Result`。上层系统**对接口编程，不关心里面是占位还是真玩法**。

### 2.2 占位战利品（测试物品）

| 占位物 | 用途 | 关键字段 |
|--------|------|---------|
| 测试食物·⚪/🟢/🔵/🟣 | 验证背包占格、变质、结算 | 稀有度、占格、变质速度（《参数总表》§3）、新鲜度档 |
| 测试钥匙 | 验证钥匙门 | 计入"已持钥匙数" |
| 测试道具 | 验证仓库 | 简单标记即可 |

### 2.3 各交互的占位行为（骨架版照此实现）

| 交互（正式玩法） | 占位行为（骨架版） | 发声(§五) | 测试选项 |
|------------------|-------------------|----------|---------|
| 🧊冰柜·敲冰 | **一个按键**：按下 → 发声 → 得 1 件占位食物（随机稀有度） | 终破 10m | 选稀有度、高噪/静音 |
| 🔑容器出钥匙 | 普通容器搜刮**按几率**直接给"测试钥匙" | 容器搜索噪音 | 滑条调几率 |
| 🔑三钥匙大门 | **一个按键**：已持 3 把 → 开门给占位高价值食物；否则提示"还差 N 把" | 开门事件（可选） | 一键补满 3 钥匙 |
| ⚡渡电 | **一个按键**：按下 → 发声 → 置"通路已开"标志 | 落水 15m | 强制开路 / 强制落水 |
| 👹饕餮遭遇·双环 | **占位界面**：两个按钮"赢 / 输" → 改背包（赢=零损失 / 输=随机丢 1~3 件占位食物） | 进入=遭遇音 | 强制赢/输、选丢几件 |

> 例（对齐你的说明）：**与冰柜互动 = 一个单纯按键，按下发出声音并得到占位战利品**，用于快速测"发声→惊动怪物""战利品→进背包→变质→结算"。

### 2.4 占位界面统一要求

- 极简 Debug 面板：当前交互名 + 可选测试参数 + 触发按钮。所有结果走 `Result` 回主世界，**不写死在交互里**。

## 三、时间系统（地基，多系统都读它）

| 项 | 内容 |
|----|------|
| 职责 | 提供**单局 15min 硬上限**、食物变质计时、各类时长（动作 / 搜索 / 撤离倒计时 / 占位 stubDelay）的统一时钟 |
| 关键数据 | **单局剩余时间（15:00 硬上限）**；每件食物"新鲜度计时器 + 当前档"；撤离倒计时（10s） |
| ⚠️ 15min 硬上限 | **正式版单局 = 15 分钟；15:00 未撤离 = 迷失（见 §十二），失去全部背包物资**。临近 15:00 难度档进入"狂暴/最后冲刺"（怪物最凶），但不再有"15min+ 继续玩" |
| 接口 / 事件 | `OnTick(dt)`；`OnRaidTimeout`（→ 触发迷失结算）；`OnFoodTierChange(food, 新档)` |
| 配置（《参数总表》） | 单局 15min §1；变质速度按稀有度 §3；撤离倒计时 §1 |
| 网络 | 单局时钟与各食物档位**主机为准**广播；客户端插值显示 |
| 自测 | 🟢食物按配置降到🟡🟠🔴；保温箱内暂停；**15:00 触发迷失**（未撤离者背包清空） |

## 五、声音范围系统 + 怪物寻人逻辑

> 详细见《饿魔感知与寻人逻辑》；此处给程序的接口与状态机。

| 项 | 内容 |
|----|------|
| 职责 | 动作在**发生位置**产生声音半径；怪物在半径内被惊动，前往**声源点**（非玩家当前位置）调查 |
| 声音事件 | `SoundEmit{ pos, radius, level }`——来源：动作（§四）、搜刮（§七）、占位交互（§二） |
| 怪物状态机 | **巡逻 → 调查(到声源搜8~12s) → 搜寻(更大范围12~15s) → 锁定(近距清晰视线~1.5s) → 吸入(3s) → 触发遭遇** |
| 关键数值 | "一次惊动 ≈15~25s **＜** 转盘 90s"（躲优先保证）；视野锥~8~10m/90°；时间难度档 T1/T2/T3 缩放（《参数总表》§8/§13.4） |
| 与占位 | 怪物 Lock 后调《转场》§十一 进入双环占位界面 |
| 配置（《参数总表》） | 声音半径表 §7（★三处共用）；感知计时器 §8 |
| 网络 | **怪物唯一、主机权威**；声音上报主机，主机跑状态机并广播 |
| 自测 | 奔跑→怪物扑**声源点**；静步走开→15~25s 放弃；视线被墙遮断 |

## 六、躲藏系统

| 项 | 内容 |
|----|------|
| 职责 | 提供躲点，让被惊动的怪物在搜寻后**放弃**；是"躲优先"的落点 |
| 功能型 | **程序只分 2 种：容量 1 人 / 容量 2 人**；柜/床底/纸箱/桌底等只是**美术模型 + 被发现概率数值**（《参数总表》§9） |
| 交互 / 规则 | 靠近按 E 进入 / 退出；躲藏中**发声≈0、遮断视线**；**无屏息条、无 QTE** |
| 被发现判定 | 仅当怪物处"搜寻"态且极近处搜到该躲点时判一次（概率为模型数值） |
| 接口 / 事件 | `EnterHide / ExitHide`；`OnDiscovered`（→ 升级锁定→遭遇） |
| 网络 | 躲藏状态同步；被发现判定**主机权威** |
| 自测 | 钻躲点能让搜寻中的怪物放弃；纸箱堆容 2 人；桌底比储物柜更易被发现 |

| 项 | 内容 |
|----|------|
| 职责 | 容器可搜，产出食物 / 钥匙 / 道具（含**占位战利品**）；搜索有读条、噪音、剪影识别；**地图按已接订单保证生成所需食物**（§十） |

## 十一、场景互动 → 小游戏转场系统

| 项 | 内容 |
|----|------|
| 职责 | 统一管理"触发交互 → 加载（占位）界面 → `Run()` → 拿 `Result` → 返回主世界并应用" |
| 触发源 | ① 主动（靠近冰柜/钥匙门/渡电按 E）；② 被动（怪物 Lock → 强制拉入双环占位，全队传送） |

| 占位重点 | 骨架版 `Run()` 直接返回占位 `Result`；**强制传送 + 返回主世界链路必须真实**（要早测的接合点，不是占位） |
| 网络 | 谁触发、是否全队卷入、`Result` 应用**主机权威广播** |
| 自测 | 冰柜占位键能进/出并把占位食物给背包；怪物 Lock 把三人拉进双环占位、出结果一起回遭遇点 |

## 十二、撤离系统（含 15min 迷失）

> **玩家无血量、无倒地、无死亡**——饿魔不杀人，只抢食物（转盘=经济损失）。**唯一"出局"方式 = 15:00 超时迷失。**

| 项 | 内容 |
|----|------|
| 职责 | 管理"出局"：成功撤离 → 结算；**15:00 超时未撤 → 迷失** |
| 撤离 | 地图 **1 个撤离点**（入口=出口）；按撤离键起 **10s 倒计时**（期间可移动但减速）；倒计时结束**带背包物资撤出 → 进结算**（§十三） |
| ⚠️ 迷失（塔/三角洲式） | **15:00 未撤离 = 迷失在战局 → 失去全部背包物资**（食物/钥匙/道具/战利品全失、不入仓库、不计结算）。这是**唯一**的败北方式 |
| 跑尸（可选 / 待定） | 参考塔/三角洲：下一局可回到迷失点回收**装备**；**背包内食物 / 战利品不可回收**（已失）。🔴 待你定是否保留跑尸 |
| 接口 / 事件 | `StartExtract / OnExtracted(背包)`；`OnRaidTimeout → 迷失` |
| 配置（《参数总表》） | 撤离倒计时 10s §1 |
| 网络 | 撤离 / 迷失**主机权威**；15:00 由主机统一判定迷失并清空相应背包 |
| 自测 | ① 撤离成功→结算有货入仓库；② **15:00 未撤→背包清空、无结算** |

## 十三、结算系统（新鲜度 × 订单完成度 × 数量）

| 项 | 内容 |
|----|------|
| 职责 | 撤离后，按**带出食物的新鲜度档**与**满足订单的食物数量**计分 → 报酬；展示"差一点"。**迷失者不参与（无带出物）** |
| 计分要素 | ① 新鲜度系数（🟢100/🟡60/🟠30/🔴0%，§4）；② 完成度=min(合格交付数, 需求数)/需求；③ 数量 |

**建议结算算法（可实现、可调）**：

```text
对每张已接订单：
  合格交付数 = 背包带出、符合该订单(类型/稀有度)的食物件数（封顶=需求数）
  完成度     = 合格交付数 / 需求数
  平均新鲜度系数 = 这些交付食物的新鲜度系数均值
  订单报酬   = 基础报酬 × 完成度 × 平均新鲜度系数 × 订单倍率(精确×2)
全局：
  品质分   = Σ(稀有度基分 × 新鲜度系数) 的均值 → 查表评级(90-100优秀×1.5…，§11)
  合作加成 = 完成1单+10% / 2单+25%
  最终报酬 = Σ订单报酬 × 品质评级系数 × (1+合作加成)
"差一点"展示 = 记录"某食物撤离前X秒降档""多搜40s害2件降档"(文案可后补)
```

| 项 | 内容 |
|----|------|
| 接口 / 事件 | `Settle(带出食物, 已接订单)`；产出 → 仓库 / 金币（§九） |
| 配置（《参数总表》） | 新鲜度系数 §4；订单 / 评级 / 合作加成 §11 |
| 网络 | 撤离后统一计算，**主机权威**，三人同一结算页 |
| 自测 | 带 3 件🟢满足"3×🟢"=高报酬；1 件变🟡→报酬降；只带 2 件→完成度 2/3；**迷失者结算为 0** |
