## 一、玩法规则

1. 冰封玻璃柜里冻着一件食物，**透过霜面能看到它的稀有度光色**（⚪🟢🔵🟣）——玩家因此大致知道"这块要砸多少下、值不值得"。
2. 玩家对准玻璃**连续点击鼠标**，**每一次点击在玻璃上砸出一道裂纹**，裂纹随点击累积、逐级扩散（蛛网化）。
3. 砸够对应次数后（⚪8 / 🟢12 / 🔵16 / 🟣20），玻璃**整体爆碎飞溅**，食物弹出可拾取。
4. **没有失败**——无论砸多久都能取出。张力来自**投入产出与噪音**：稀有度越高 = 玻璃越厚 = 砸得越久越吵 = 越招饿魔，形成"**要不要为这块紫货冒险砸满 20 下**"的取舍。

> **设计要点（爽感）**：核心快感来自"逐级敲碎"的**渐进破坏反馈**——每一击都要让裂纹更密、声音更脆、即将破的预兆更强，最后一击爆碎释放。**8~20 下不是为了拖时间，而是把"碎裂"拉成一条上升的爽感曲线**；调不出递增手感这个小游戏就不成立（🔴 人主导）。

> **与旧版的区别**：取消"擦霜窥视 + 多冰格单选盲盒 + 未选格报废"。现在是**单件食物 + 连点砸碎**，用稀有度决定厚度（点击数）。

## 二、流程与状态机

```text
[待机 Idle] ──玩家对准开始点击──▶ [敲砸 Smashing] ──每击累积裂纹·分级扩散──┐
                                       │  点击数 < 阈值，继续砸 ↺              │
                                       └──────────────────────────────────────┘
                                                  │ 点击数达标(8/12/16/20)
                                                  ▼
                                        [爆碎 Shatter(瞬时)] ──▶ [结算 Resolve] ──▶ [完成 Done]
                                                                  食物弹出可拾取
```

| 状态 | 行为 | 进入 | 退出 |
|------|------|------|------|
| Idle | 玻璃覆霜、透出稀有度光色、可交互高亮 | 默认 | 玩家点击 |
| Smashing | 累积点击、裂纹分级扩散、音画递增 | 首次点击 | 点击数达阈值 |
| Shatter | 玻璃爆碎飞溅（瞬时，大 punch + 震屏） | 达阈值 | 立即转 Resolve |
| Resolve | 食物弹出/入背包、终破声音半径触发 | Shatter | 完成 |
| Done | 柜内取空 | Resolve | 回主循环 |

> **中断**：玩家中途走开（饿魔逼近/撤离），已积累裂纹**保留 N 秒**（可调）再缓慢回冻；回来可接着砸。允许"砸一半先躲，回头再砸"。

## 三、时长预期（详细 · 随稀有度伸缩）

> 参考手速 **~3 击/秒**（实际因人而异）；另加进入/弹出约 1~1.5s overhead。

| 食物稀有度 | 点击数 | 砸玻璃耗时 | 含 overhead 总时长 |
|-----------|-------|-----------|-------------------|
| ⚪ 普通 | 8 | ~2.5~3 s | ~3.5~4 s |
| 🟢 优质 | 12 | ~4 s | ~5 s |
| 🔵 稀有 | 16 | ~5~5.5 s | ~6.5 s |
| 🟣 传说 | 20 | ~6.5~7 s | ~8~9 s |
| 🟡 神话（若冰封） | 待定（≥20 或特殊） | — | 🔴 待拍板 |

- **合作显著缩短**：多人同砸点击数共享，2~3 人 ≈ 单人的 1/2~1/3 时长（见 §八）。
- 这条"稀有度→时长/噪音"的伸缩，是把高价值食物天然变成"高风险目标"的关键。

## 四、可配置参数表（程序读表、不写死）

| 参数 | v1 值 | 说明 |
|------|-------|------|
| 各稀有度点击数 | ⚪8 / 🟢12 / 🔵16 / 🟣20 | 用户定；🔴 灰盒验"20 下是否太累" |
| 神话级点击数（若冰封） | 待定 | 建议 ≥20 或做特殊处理 |
| 裂纹分级数 | 4 级（每 25% 进度一档视觉） | 渐进爽感的可读进度 |
| 单击裂纹声音半径 | ~5~6 m | 连续砸=持续噪音，接感知系统 |
| 终破碎裂声音半径 | **10 m** | 最后爆碎的大响 |
| 合作点击是否共享计数 | 是 | 多人同砸汇入同一计数 |
| 中断后裂纹保留时长 | ~5 s 后回冻 | 支持"砸一半先躲" |
| 稀有度光色是否透显 | 是 | 关=盲砸惊喜（可灰盒对比） |
| 镜头 punch / 震屏递增 | 人调 | 🔴 爽感 |

## 五、镜头设计

| 阶段 | 镜头行为 |
|------|---------|
| 进入 | 0.4s lerp 到 ~45° 斜俯视，聚焦玻璃柜正面、透显食物光色 |
| 敲砸（递增） | 每击一个**微 punch + 裂纹高亮**；随裂纹累积，镜头**轻微越拉越紧**（拉近/震感逐级上升），制造"快破了"的预兆 |
| 爆碎 | **大 punch-in + 震屏 + 顿帧**，飞溅朝镜头——爽感释放峰值 |
| 结算 | 0.5s 看清飞溅与食物弹出，再回俯视 |
| 三人同砸 | 镜头框住玻璃 + 多名角色围砸（party 混乱感）；不强制锁单人 |

**要点**：镜头紧绷度要随裂纹进度上升、在爆碎一刻彻底释放——这是"渐进爽感"的镜头语言，🔴 人手调。

## 六、美术风格与资产清单

**风格基调**：美食废土 × 低模卡通；裂纹进度与稀有度光色必须俯视一眼可读。

| 资产 | 说明 | 俯视可读重点 | 优先级 |
|------|------|-------------|--------|
| 冰封玻璃柜 | 锈框 + 霜面玻璃，**透出食物稀有度光色** | 光色醒目（知道是几下的货） | 🔴 必做 |
| **裂纹分级**（爽感核心资产） | 4 级蛛网裂纹贴图/网格，逐级扩散、变白 | 进度像"裂纹血条"一眼可读 | 🔴 必做 |
| 爆碎飞溅 | 低多边形冰晶 + 玻璃碎 + 寒气白雾 + 食物弹出 | 飞溅朝镜头、释放感足 | 🔴 必做 |
| 交互提示 | "点击砸冰"图标 + 可交互高亮 | 远处可见 | 🟡 |

> **音画递增是这个小游戏的命脉**：每一击的裂纹+碎冰声+(可选)手柄震动必须逐级加强，把 8~20 下做成"上升—爆发"曲线，而非重复劳动。

## 七、声音接口（接《感知与寻人》系统）

| 事件 | 声音半径 | 噪音级 | 说明 |
|------|---------|--------|------|
| 单击裂纹 | ~5~6 m | 中 | **连续砸 = 持续脉冲噪音**；砸得越久（越稀有）暴露越久 |
| 终破爆碎 | **10 m** | 中偏高 | 最后一击的大响 |

- **静音胶带**：本次敲冰全程声音 ×0.4（或封顶 4m）——可安全砸高价值厚玻璃。
- **风险设计落点**：高稀有 = 砸 20 下 = 长时间 5~6m 噪音 + 一记 10m 终响 → 天然把"贪紫货"和"招饿魔"绑定。建议在砸击时持续显示声波环。

---

## 八、三人合作价值

- **多人同砸一块玻璃**：各人的点击**汇入同一计数**，3 人砸 20 下的紫货 ≈ 每人 7 下，飞快。
- 自然喊话："一起砸那块紫的！""你俩来这边，我去望风！"
- party 混乱爽感 = 三个人对着一块玻璃疯狂点击、碎裂齐飞。
- 合作把"高稀有=久砸=久暴露"的风险压下来——**抢大货时凑人头砸**是最优解。

## 九、失败 / 代价反馈

| 机制 | 表现 |
|------|------|
| **无失败态** | 砸够次数必出食物 |
| 唯一代价 | **时间 + 噪音**：稀有度越高砸越久、越吵、越招饿魔 |
| 心理落点 | 爆碎一刻的释放爽感 + "为这口冒了多大险"的回味 |

## 十二、灰盒验证清单

1. **连续砸碎的爽感成立吗**？逐级裂纹 + 终破是否"解压"而非重复劳动？
2. **8/12/16/20 的投入感合理吗**？尤其 **🟣20 下会不会太久/手酸**？（重点：单人砸 20 下的体感）
3. 高稀有"久砸久吵"的噪音风险，是否被玩家感知为"值不值得冒险砸"？
4. 多人同砸是否好笑、且明显更快？
5. 裂纹进度（像血条）是否一眼可读？

---

## 十三、白盒最低形态（可先于美术）

玻璃柜 + 透显稀有度光色 + 点击计数 + 裂纹分级占位（4 级）+ 爆碎占位 + 食物弹出。**不需要**：精细裂纹 shader、寒气特效、最终音效。**重点先验**：单人砸 🟣20 下的体感（太累就调数值或加合作权重）。
