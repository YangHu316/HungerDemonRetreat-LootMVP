## 一、玩法规则

1. **多格冰柜 + 毛玻璃**：冰柜分 **1×1 / 2×2 / 3×3** 个冰格，每格冻着一件食物（稀有度**按概率随机**，见 §四）。柜门初始为**毛玻璃，看不清内容**。
2. **擦玻璃（每柜 1 次、只能擦 1 格）**：玩家可**擦开 1 格**看清其内容（含名称）；其余格保持毛玻璃、靠赌——有限的"窥视"资源。
3. **连点砸碎、可随时改格**：对某格**连续点击鼠标**逐步敲碎玻璃，点击数随该格稀有度（⚪4/🟢6/🔵10/🟣15/🟡20）。**破碎前可随时改敲别格**——各格**独立累计**进度，先砸碎的那格即为所选格。
4. **内容物 + 名称**：内容物用**图片**显示；**名称悬浮在其上方，颜色＝品质色**。砸开后**所有格名称揭示**，让玩家清楚"选了什么、舍弃了什么"（如舍弃了金色佛跳墙）。
5. **砸开 → 其余格变质报废**：某格砸碎后食物**留在格子内**（移到靠前、方便看清），不弹出；**其余格食物立即变质失效（发黑 + 绿烟 + 变透明）**，无法再取——**盲盒选择的代价**。
6. **没有失败**——砸够必出。张力＝**选择代价**（舍弃了哪些格、可能舍弃了金货）＋**投入与噪音**（越稀有越厚＝砸越久越吵）。

> **设计要点**：① "**选哪格的犹豫**"——毛玻璃下只能擦看 1 格、其余靠赌；赌金货（20 击慢而吵）还是稳拿易货？舍弃的都会废。② "**渐进碎裂爽感**"——每击裂纹更密、声音更脆、爆碎释放；**界面不显示剩余敲击数，靠裂纹大小 + 屏幕抖动判断进度**。两者都是命脉（🔴 人主导）。

> **与历史版的关系**：本版＝最初的"多冰格盲盒 + 未选格报废" ＋ "擦霜窥视"（现为**擦玻璃**）＋ "连点砸碎（点击数随稀有度）"的**合并**。多格盲盒选择**始终保留**。

## 二、流程与状态机

```text
[选格 SELECT] ──玩家首击某格→锁定该格──▶ [敲砸 SMASHING] ──每击累积裂纹·分级扩散──┐
   （多格透色，看哪格值得）                  │  点击数 < 阈值，继续砸 ↺              │
                                            └──────────────────────────────────────┘
                                                       │ 点击数达标(4/6/10/15/20)
                                                       ▼
                            [爆碎 SHATTER] ──▶ [结算 RESOLVE：食物留格内 ＋ 其余格变质] ──▶ [完成 DONE]
```

| 状态 | 行为 | 进入 | 退出 |
|------|------|------|------|
| SELECT(IDLE) | 多格透出光色、可交互高亮、等待选格 | 默认 | 玩家首击某格 → 锁定 |
| SMASHING | 仅所选格累积点击、裂纹分级、音画递增 | 锁定后 | 点击数达阈值 |
| SHATTER | 所选格玻璃爆碎飞溅（瞬时，大 punch + 震屏 + 顿帧） | 达阈值 | 立即转 RESOLVE |
| RESOLVE | 所选格食物**留在格内显示**；**其余格食物变质报废（发黑+绿烟）** | SHATTER | 完成 |
| DONE | 取出所选格食物，回主循环 | RESOLVE | — |

> **随时改格 / 中断**：**各格独立累计**进度，**破碎前可随时改敲别格**（点哪格砸哪格）。停手时该格裂纹**保留 N 秒**再缓慢回冻。允许"砸一半先躲，回头再砸 / 改砸别格"。

## 三、时长预期（详细 · 随所选格稀有度伸缩）

> 参考手速 **~3 击/秒**；另加选格犹豫 + 进入/弹出约 1~2s。

| 所选格稀有度 | 点击数 | 砸玻璃耗时 | 含 overhead 总时长 |
|-----------|-------|-----------|-------------------|
| ⚪ 普通 | 4 | ~1.5 s | ~2~3 s |
| 🟢 优质 | 6 | ~2 s | ~3~3.5 s |
| 🔵 稀有 | 10 | ~3.5 s | ~4.5~5 s |
| 🟣 传说 | 15 | ~5 s | ~6~7 s |
| 🟡 神话 | 20 | ~6.5 s | ~7.5~8.5 s |

- **合作显著缩短**：多人同砸所选格，点击共享，2~3 人 ≈ 单人的 1/2~1/3。
- 格数越多 → 选格犹豫越久、放弃的越多（3×3 砸 1 格 = 报废 8 格），选择张力越强。

## 四、可配置参数表（程序读表、不写死）

| 参数 | v1 值 | 说明 |
|------|-------|------|
| **冰柜大小** | 1×1 / 2×2 / 3×3 | 可固定或随机；格数越多选择代价越大 |
| 各稀有度点击数 | ⚪4 / 🟢6 / 🔵10 / 🟣15 / 🟡20 | 越稀有玻璃越厚 |
| 每格稀有度 | 随机 | 由全局稀有度分布决定（高危区高罕率↑） |
| **稀有度生成概率** | ⚪40% / 🟢30% / 🔵20% / 🟣9% / 🟡1% | 基础权重；再按区域危险度加权 |
| 每柜擦玻璃次数 | 1（只能擦 1 格） | 有限窥视资源 |
| 内容物显示 | 图片 + 名称悬浮（颜色＝品质色） | 砸开后全揭示 |
| 剩余敲击数显示 | **不显示** | 只靠裂纹大小 + 屏幕抖动判断 |
| 信息揭示 | **毛玻璃（默认看不清）+ 每柜擦 1 格揭示** | 决定盲盒"有多盲" |
| 裂纹分级数 | 4 级（每 25% 一档） | 渐进爽感可读进度 |
| 单击裂纹声音半径 | ~5~6 m | 连续砸=持续噪音 |
| 终破碎裂声音半径 | **10 m** | 最后爆碎的大响 |
| 合作点击共享计数 | 是 | 多人同砸汇入同一计数 |
| 中断后裂纹保留 | ~5 s 后回冻（各格独立） | 破碎前可随时改格、各格独立累计 |
| 镜头 punch / 震屏递增 | 人调 | 🔴 爽感 |

**内容物命名库（v1）**：
- ⚪ 白：米饭 / 馒头 / 面包 / 地瓜 / 玉米 / 挂面
- 🟢 绿：包子 / 饺子 / 炒饭 / 蘑菇汤
- 🔵 蓝：汉堡 / 盖饭 / 水果蛋糕
- 🟣 紫：黯然销魂饭 / 烧鹅
- 🟡 金：佛跳墙

> 每格按稀有度从对应列表随机取名；图片放 `assets/food/<名称>.jpg`（缺图自动用纯色占位）。已在 `Godot_Demo` 实现。

## 五、镜头设计

| 阶段 | 镜头行为 |
|------|---------|
| 进入 / 选格 | 0.4s lerp 到斜俯视，框住整个多格冰柜正面、透显各格光色（便于选） |
| 敲砸（递增） | 每击微 punch + 裂纹高亮；随裂纹累积镜头**越拉越紧**，制造"快破了"的预兆 |
| 爆碎 | **大 punch-in + 震屏 + 顿帧**，飞溅朝镜头——释放峰值 |
| 结算 | 看清所选格食物弹出 ＋ 其余格变质冒烟，再回常规 |
| 三人同砸 | 镜头框住冰柜 + 多人围砸 |

## 七、声音接口（接《感知与寻人》系统）

| 事件 | 声音半径 | 噪音级 | 说明 |
|------|---------|--------|------|
| 单击裂纹 | ~5~6 m | 中 | 连续砸=持续脉冲噪音；砸越久暴露越久 |
| 终破爆碎 | **10 m** | 中偏高 | 最后一击的大响 |

- **静音胶带**：本次敲冰全程声音 ×0.4（或封顶 4m）。
- 高稀有 = 砸 15~20 下 = 长噪音 + 终响 → 把"贪大货"和"招饿魔"绑定。

## 八、三人合作价值

- **多人同砸所选格**：点击汇入同一计数，3 人砸金货 ≈ 每人 7 下。
- **共同选格**：帮忙看哪格值、要不要赌金货（其余都会废）→ 喊话："砸金的！""那个绿的稳，别贪！"
- party 混乱爽感 + 选择争论。

## 九、失败 / 代价反馈

| 机制 | 表现 |
|------|------|
| **无失败态** | 砸够所选格必出食物 |
| **选择代价（核心）** | **其余未选格食物立即变质报废**（发黑 + 绿烟）——选了一格就放弃别的 |
| 时间 / 噪音代价 | 稀有度越高砸越久、越吵、越招饿魔 |
| 心理落点 | "敲对了这格吗？放弃的那格会不会更值"——盲盒惜败，服务"差一点"续局 |

## 十二、灰盒验证清单

1. **"选哪格的犹豫感"成立吗**？多格里有高有低，玩家会不会纠结、和队友争论？
2. **其余格变质的"心疼/代价"是否到位**？放弃的格废掉是否让玩家在意？
3. **连续砸碎的爽感成立吗**？逐级裂纹 + 终破是否"解压"？
4. **4/6/10/15/20 的投入感合理吗**？尤其 🟣15 / 🟡20 下单人体感。
5. 裂纹进度（像血条）是否一眼可读？玻璃碎裂时裂纹是否随之消失（不悬浮）？

## 十三、白盒最低形态（可先于美术）

多格冰柜（1×1/2×2/3×3）+ 每格透色 + 选格锁定 + 点击计数 + 裂纹分级占位（4 级）+ 爆碎占位 + 食物弹出 + **其余格变质（发黑/绿烟占位）**。**不需要**：精细裂纹 shader、寒气特效、最终音效。**已在 `Godot_Demo/eft-delivery` 实现并验证**（含 1×1/2×2/3×3、选一格、其余变质、裂纹随玻璃消失）。
