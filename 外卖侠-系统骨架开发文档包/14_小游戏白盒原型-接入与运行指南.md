## 一、机制速览（含操作 / finished 返回 / 美术槽）

### 🧊 敲冰开柜（`ice_cabinet`，Node3D）
- **机制**：冰柜分 1×1 / 2×2 / 3×3，每格按概率（白40/绿30/蓝20/紫9/金1）放一件随机稀有度食物，初始毛玻璃看不清。每柜 1 次「擦玻璃」看清 1 格；玩家选 1 格连点砸碎（点击数随稀有度 4/6/10/15/20）取出；砸开后其余格食物变质报废。**无失败**。
- **操作**：左键=砸冰（首击锁定该格）/ 右键=擦玻璃（1 次）/ `[1·2·3=改冰柜大小, R=重置 ← 仅开发者模式]`
- **finished**：`{ "won": true, "food": <名称String>, "rarity": <int 0~4> }` → 取出 1 件食物
- **美术槽**：`cabinet_model`(PackedScene 外壳) / `food_models`(Dictionary 名称→PackedScene) / `shard_model`(碎片)

### ⚡ 凌波渡电（`electric_crossing`，Node3D）
- **机制**：通电水面 + 落脚点。长按蓄力跳抛物线落到下一落脚点（蓄力 1.2s→跳距 0.8~2.6m）；落水=触电弹回损失 ~2.2s；到对岸端拉电闸断电（1.2s），随后趟水过去。**这是个「通行闸」而非搜刮**。
- **操作**：长按 `空格`=蓄力跳 / `E`=拉电闸（到对岸端时）/ `[R=重置 ← 仅开发者模式]`
- **finished**：`{ "crossed": true }` → 过池成功（**不是战利品，是通行**）
- **美术槽**：`stone_model`/`bank_model`/`player_model`/`follower_model`/`switch_base_model`/`switch_lever_model` + `water_material`
- **发声**：落水/电弧声半径 15m（`@export sound_radius_splash`），应接饿魔感知（见 §3.5）

### 🌀 暴食传送门（`gluttony_portal`，**Node2D**）
- **机制**：俄罗斯方块＋合成大西瓜的背包。每局掉落 N=14 件（⚪🟢🔵）；同稀有度成簇合成 **2🔵→🟣 / 3🟣→🟡**（🟡封顶，紫/金不掉落只能合）；夹杂变质堵块（概率 0.18，上限 5）。容器溢出=撕裂→食物**散落一地**→7s 抢救窗口（按 F 把高价值塞进包，或弃物逃走）。
- **操作**：`←→ / A D`=移动，`↑ / W`=旋转，`↓ / S`=软降，`空格`=硬降 / `[散落态 F=抢救, R=重置 ← 仅开发者模式]`
- **finished**：
  - 胜利（装满入容器）：`{ "won": true, "loot": [<名称String>...] }` → 把这批食物给玩家
  - 撕裂结束：`{ "won": false, "fled": <bool>, "bagged_value": <int>, "bagged_count": <int> }` → 抢救进包的聚合
- **美术槽**：`food_textures`(Dictionary int稀有度→Texture2D) / `portal_texture`

## 二、怎么把它跑起来（三种方式）

1. **沙盒试玩（最快）**：在 FileSystem 里 F6 运行 `eft_greybox/dev_launcher.tscn` → 自动以 **standalone=true**（自带相机/光照/地面/自动开始）加载第一个小游戏；按 **F2 循环切换**三个小游戏。
2. **单场景 F6**：选中某个小游戏 `.tscn` 的根节点，Inspector 把 `standalone` 勾 **true**，F6 直接跑。⚠️ **跑完取消勾选**——接入大地图时必须 `standalone=false`（嵌入态不抢相机、不自动开始）。
3. **开发者测试键（可选）**：测试键（`R`重置、敲冰 `1/2/3` 改尺寸等）受全局 `Dev` 开发者模式门控。要启用需在 `project.godot` 的 `[autoload]` 加一行：
   ```
   Dev="*res://eft_greybox/dev_mode.gd"
   ```
   注册后按 **F1** 开/关开发者模式（屏幕右上角显示「● 开发者模式 ON」）。**不注册也能正常玩**（鼠标/空格/方向键玩法键不受影响），只是测试键全程禁用、小游戏对 `/root/Dev` 有空守卫不会报错。

## 三、接入「测试地图的场景交互」（核心）

### 3.1 先理解你们现有的交互协议
- 输入动作 `interact` = **F**。
- **交互物**：进 `interactables` 组 + 实现四件套 `get_interact_position()` / `get_prompt()` / `is_available()` / `interact(player)`；用子 `Area3D` 在玩家进入时调 `player.register_interactable(self)`、离开时 `unregister_interactable(self)`。`player.gd` 每帧选最近可用交互物，显示其 `get_prompt()`。（参考实现：`scripts/entities/container.gd`）
- **按 F 时的派发现状（关键）**：
  - `player.gd`：**只对 `doors` 组**的最近交互物调 `interact(self)`。
  - `main.gd`：对 `local_player.nearby_container`（容器）开搜刮 UI。
  - ⇒ **其它交互物的 `interact()` 目前没人自动调** → 小游戏要接入，需要给它一个派发入口（见下，仅 1 行改动）。

### 3.2 推荐做法：MinigameStation 小游戏站（照搬 container 结构）
在地图里摆一个交互物当「小游戏入口」（外观可直接用 `eft_greybox/kit/kit_minigame_socket.tscn`），挂下面的脚本。它做四件事：**进组 → 实现协议 → interact 启动小游戏 + 锁玩家 → finished 解锁 + 发战利品**。

> **唯一需要改的现有代码（1 行）**：在 `scripts/entities/player.gd` 的 interact 分支里，把
> `if _nearest.is_in_group("doors"):` 扩成
> `if _nearest.is_in_group("doors") or _nearest.is_in_group("stations"):`
> 让最近的小游戏站被按 F 时也收到 `interact(self)`。（按你们 7 步闭环，给它补一条 GUT 守一下这个分支。）

```gdscript
extends Node3D
## 小游戏站：放进地图的交互物，按 F 启动一个小游戏，结束后把战利品给玩家。
## 站点场景需含一个子节点 Trigger:Area3D(带 CollisionShape3D, 半径~1.5m)——照搬 container.tscn 的 Trigger。

@export var game_scene: PackedScene          # 例：res://eft_greybox/ice_cabinet.tscn
@export var prompt_text: String = "按 F 互动"
@export var overlay_2d: bool = false         # 暴食(Node2D)勾 true → 全屏覆盖；敲冰/渡电留 false(世界物)

var _busy: bool = false
var _game: Node = null
var _overlay_layer: CanvasLayer = null

func _ready() -> void:
	add_to_group("interactables")
	add_to_group("stations")                 # ← 配合 player.gd 的一行挂钩
	var trigger: Area3D = $Trigger
	trigger.body_entered.connect(_on_body_entered)
	trigger.body_exited.connect(_on_body_exited)

# ── interactables 协议四件套 ──
func get_interact_position() -> Vector3: return global_position
func get_prompt() -> String: return prompt_text
func is_available() -> bool: return not _busy

func interact(player: Node) -> void:
	if _busy or game_scene == null:
		return
	_busy = true
	if player != null:
		player.movement_locked = true        # 交出控制权：冻结玩家移动
	_game = game_scene.instantiate()
	_game.set("standalone", false)           # 嵌入态：用宿主相机、不自动开始
	if overlay_2d:
		_overlay_layer = CanvasLayer.new()   # 2D 小游戏 → 全屏覆盖(不放进世界)
		add_child(_overlay_layer)
		_overlay_layer.add_child(_game)
	else:
		add_child(_game)                     # 3D 小游戏 → 世界里(站点处)
	_game.connect("finished", Callable(self, "_on_finished").bind(player))
	_game.call("begin")

func _on_finished(result: Dictionary, player: Node) -> void:
	if player != null:
		player.movement_locked = false       # 收回：解锁玩家
	_grant_loot(result)                      # ↓ 按你们背包 API 实现(见 §3.4)
	if _overlay_layer != null:
		_overlay_layer.queue_free(); _overlay_layer = null
	elif _game != null:
		_game.queue_free()
	_game = null
	_busy = false

func _grant_loot(result: Dictionary) -> void:
	if not bool(result.get("won", false)):
		return                               # 渡电(crossed)/暴食失败：在此另作处理(见 §3.4)
	var names: Array = []
	if result.has("loot"):                   # 暴食胜利：名称数组
		names = result["loot"]
	elif result.has("food"):                 # 敲冰：单件名称
		names = [result["food"]]
	for nm in names:
		var item: ItemData = _item_by_name(String(nm))   # ← TODO 你们实现 名称→ItemData
		if item == null:
			continue
		# 空位用你们的 GridPlacer.find_first_fit(<玩家背包grid>, item)，再 place_entry：
		var entry: Dictionary = { "item": item, "x": 0, "y": 0, "rotated": false,
			"freshness_elapsed": 0.0, "uid": -1 }
		PlayerInventory.place_entry(entry, 0, 0)   # 见 §3.4 注意事项

func _item_by_name(_nm: String) -> ItemData:
	# 现无 名称→ItemData 注册表(见 §3.4)。约定「名称＝item id」时可直接：
	#   return load("res://resources/items/%s.tres" % _nm) as ItemData
	return null

func _on_body_entered(body: Node) -> void:
	if body.is_in_group("player") and body.has_method("register_interactable"):
		body.register_interactable(self)
func _on_body_exited(body: Node) -> void:
	if body.is_in_group("player") and body.has_method("unregister_interactable"):
		body.unregister_interactable(self)
```

### 3.3 3D 世界物 vs 2D 覆盖层（别接错）
- **敲冰 / 渡电（Node3D）**：作为世界道具，`add_child` 到站点处，用现有俯视相机，玩家鼠标/键直接操作。需要时可把相机推近到取景位（见 `eft_greybox/接入大地图与开发者模式-说明.md` §一）。
- **暴食（Node2D）**：当成**全屏 2D 覆盖**——实例化到一个 `CanvasLayer`（或 `SubViewportContainer`）下，**不是放进 3D 世界**。interact 时显示、finished 时销毁。上面脚本的 `overlay_2d=true` 已处理。

### 3.4 finished 结果 → 战利品 / 状态 映射
| 小游戏 | result | 怎么处理 |
|--------|--------|---------|
| 敲冰 | `{won, food, rarity}` | `food` 名称 → ItemData → 入背包 |
| 暴食·胜 | `{won:true, loot:[名称…]}` | 逐个名称 → ItemData → 入背包 |
| 暴食·撕裂 | `{won:false, fled, bagged_value, bagged_count}` | `bagged_*` 是已抢救进包的聚合：可直接结算价值；想要具体物品就改 `gluttony_portal_game.gd` 让它也返回 loot 列表 |
| 渡电 | `{crossed:true}` | **不是战利品**：把玩家挪到对岸 / 标记此闸已过即可 |

- **名称→ItemData 需要你们补一个查找**（现无注册表）：item 资源在 `resources/items/*.tres`，字段 `id / display_name / value / grid_w / grid_h`。三选一：① 约定「food 名称＝item id」→ `load("res://resources/items/%s.tres" % name)`；② 启动时扫 `resources/items/` 建 `display_name→ItemData` 字典；③ 在小游戏侧直接产出 item id 而非中文名。
- **入背包**：确认存在的入口是 `PlayerInventory.place_entry(entry, x, y) -> bool`（autoload 转发到本地玩家 `InventoryComp`）。`entry` 形如容器的 `{uid, item:ItemData, x, y, rotated, freshness_elapsed, …}`；空位坐标用你们 `GridPlacer.find_first_fit(<背包grid>, item)` 求（与 `container.gd._generate_contents` 同套路），别硬写 `(0,0)`。背包满时自行决定丢地上/提示。

### 3.5 控制权交接 & 多人 & 发声
- **交接**：进游戏 `player.movement_locked = true`（你们 player.gd 已有此字段，冻结移动、躲藏中也用它）；出游戏复位 `false`。
- **多人**：三个白盒目前是**单人本地**玩法。联机要走 host 权威（谁触发、结果谁广播、loot 进谁的包），参照你们容器/取物的 Phase 2B RPC 套路；MVP 阶段可先限定「单人 / 本地各玩各的」。
- **发声接入饿魔感知**：渡电落水 15m、敲冰搜刮（≈容器中噪 8m 量级）应接 `EventBus.sound_emitted.emit(global_position, radius)`（你们 §06 Phase 3B 已有此信号），让饿魔被吸引——与 `05_饿魔感知与寻人逻辑` 一致。
