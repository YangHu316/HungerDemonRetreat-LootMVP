## 一、玩法 / 系统规则

1. **钥匙是一种特殊战利品**：每局**每种钥匙保底刷新 1 把**（共 **3 种 = 3 把**），藏在**随机容器**里。玩家搜订单食物的同时，**顺带期盼开到钥匙**——"这箱会不会有钥匙？"（保底保证一定能凑齐，挑战在于"在时限内找全"）
2. 地图上有 **1 道大门**（**年久失修、开启会发出巨响**），门上 **3 个钥匙槽**，需集齐 **3 把（3 种各 1）**才能打开。门后是**高价值区 / 房间**（v7.x 半开放地图中，即重置入商业区的隐藏房间 / 🟣 食物 / 渡电区一带）。
3. 钥匙可由任意玩家搜到、**可在队友间传递、带到门前汇集**；集齐 3 把后在门前交互打开。
4. **无失败**：要么凑齐 3 把开门、要么没凑齐开不了门——纯粹是"有 / 没有"。**没有匹配推理、没有钥匙断裂、没有锁死。**

> **为什么这样改**：把"开锁"从一个**独立单元玩法**，变成**嵌入搜刮的合作收集目标**（像三角洲/塔科夫"找钥匙开房")。它不打断主循环，而是给搜刮**加一层"开盲盒式的期待"和"全队凑齐"的合作牵引**。这也成为 v7.5 里"上锁"型高价值区的解锁方式（取代旧"上锁储藏室"的匹配解障）。

---

## 二、流程与状态机

```text
搜刮主循环中：[搜容器] ──几率命中──▶ 出钥匙 ──玩家拾取持有──┐
                                                            │ 分头继续搜 / 传递 / 送到门
                                                            ▼
                                          [大门 Locked: 已插 0~2 把]
                                                            │ 集齐第 3 把、门前按 E
                                                            ▼
                                          [大门 Open] ──▶ 高价值区开放（搜刮 / 解障 / 撤离）
```

| 大门状态 | 条件 | 表现 |
|---------|------|------|
| Locked(0~2) | 钥匙不足 3 | 门上钥匙槽显示**已插几把**（进度可读） |
| Open | 集齐 3 把并交互 | 沉重开门，门后高价值区揭示 |

## 三、时长预期

- **不设独立计时**：钥匙获取取决于搜刮运气与刷新几率，**融入整局搜刮节奏**。
- 设计目标（可调）：让几率落在"**稳健~贪心打法一局内大概率能凑齐 3 把、但不保证**"——既有期待，又有"差一把"的不甘（服务"差一点"续局驱动）。
- 开门本身：一次短交互（约 1~2s，可选做成"开门瞬间高噪、最危险"的张力点，见 §七）。

## 四、可配置参数表（程序读表、不写死）

| 参数 | v1 值 | 说明 |
|------|-------|------|
| 开门所需钥匙数 | **3**（3 种各 1） | 用户定 |
| 钥匙保底刷新 | **每局每种 1 把（共 3 把），随机容器** | 用户定；保底保证可凑齐 |
| 额外刷新（保底之外） | 默认 0（仅保底 3 把） | 🔴 可选：是否再加随机多刷降低搜寻难度 |
| 可刷钥匙的容器 / 区域 | 全图随机容器（可加权商业区） | 控制钥匙分布 |
| 钥匙可否传递 | 是（复用合作传递接口） | 支持汇集 |
| 门后奖励 | 高价值区（隐藏房间 / 🟣） | 🔴 平衡"值不值得收集" |
| 开门交互时长 | 1~2 s | |
| 开门 = 高噪事件 | **是**（年久失修大门巨响，~12~15m，引饿魔） | 制造"开门一刻最危险"的张力 |

|------|---------|

## 六、美术风格与资产清单

| 资产 | 说明 | 俯视可读重点 | 优先级 |
|------|------|-------------|--------|
| 钥匙（战利品） | 醒目的旧钥匙图标 / 低模，**与食物明显区分** | 搜出时一眼认出"这是钥匙" | 🔴 必做 |
| 大门 + 3 钥匙槽 | **年久失修的破旧大门**（开启吱呀巨响），槽位显示**已插几把**进度 | 进度一眼可读、"还差几把"、破败感 | 🔴 必做 |
| 门后高价值区 | 门缝/开启后瞥见的诱惑（光、紫货微光、解障区） | "值得为它收集"的吸引力 | 🟡 |
| 开门演出 | 沉重大门开启 | 成就感 | 🟡 |

> 三把钥匙**外观可一致或各异**（各异可增"还差红色那把"的可读性，二选一，灰盒定）。

## 七、声音接口（接《感知与寻人》系统）

- **搜钥匙**：走**容器搜索噪音**（已在声音半径表，按容器类型 4~8m）——钥匙是容器 loot，不另设"翻找钥匙"噪音。
- **开门 = 高噪事件**：年久失修的大门开启时发出**巨响**（~12~15m 高噪），"开门那一刻最容易招饿魔"——要不要望风、要不要先清场、谁来开，成为合作决策。
- 旧版的"钥匙·翻找 6m / 开锁 3m"独立条目**作废**（已无独立搜钥匙/匹配动作）。

## 八、三人合作价值

- **分头搜刮**覆盖更多容器 → 提高全队**总出钥匙率**（三人比一人更快凑齐）。
- **钥匙汇集**：谁搜到带到门 / 传给跑门的人——"我搜到一把！""还差一把！""谁去门那边等？"
- **开门望风**：若开门是高噪事件，需有人放哨 / 引怪。
- 这是天然、好懂的合作目标（像三角洲组队找钥匙开房），不需要复杂规则。

## 九、失败 / 代价反馈

| 机制 | 说明 |
|------|------|
| **无失败** | 只有"凑齐 3 把 / 没凑齐"两态 |
| 没凑齐 | 门不开 = 拿不到门后高价值货 = **机会损失**（非惩罚），服务"差一把就能开"的不甘 |

## 十、网络同步要点（局域网三人）

| 项 | 处理 |
|----|------|

| 风险 | 中——刷新权威 + 门状态一致，3 真机抽测 |

## 十二、灰盒验证清单

1. **"边搜边盼钥匙"的期待感成立吗**？玩家会不会每开一个箱都期待出钥匙？
2. 三人是否自然**分头搜 + 汇集钥匙 + 喊话**？
3. **保底 3 把 + 随机容器**的搜寻强度是否恰好——在时限内找全有挑战但不至于找不到？是否需要额外多刷？
4. 开门那刻是否有"终于！"的爽点？（若做高噪，是否够紧张）
5. 门后奖励是否**值得**这趟收集（不值=没人去凑）？

---

## 十三、白盒最低形态（可先于美术）

钥匙作为容器 loot（几率出）占位 + 拾取/持有/传递 + 大门 3 槽 + 集齐判定 + 开门 + 门后高价值区占位。**不需要**：精细钥匙/大门模型、开门动画、最终音效。**重点先验**：刷新几率调到"一局大概率凑齐但有'差一把'风险"的手感。
