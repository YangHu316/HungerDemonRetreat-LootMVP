# 《饿魔退散！外卖侠》饿魔感知与寻人逻辑

> 系列文档之一。本文档把你提出的核心想法——**"范围化的声音：玩家动作附带一个声音范围，敌人若在范围内则开始搜寻玩家"**——设计成一套完整、可实现、与"躲优先"一致的感知系统。
>
> 配套阅读：《玩家躲避系统》（同一套猫鼠对抗的另一半）、《场景互动小游戏全集》（各动作的声音半径）。

---

## 〇、设计目标

| 目标 | 说明 |
|------|------|
| 让"声音"成为玩家在世界里留下的信号 | 每个动作都有声音半径；越想搜得快/多，越容易暴露 |
| 让"躲 = 最优解"成立 | 饿魔被惊动后只会去**声源点**调查，玩家安静走开/躲起来就能让它放弃 |
| 时间账必须算得过来 | 被惊动一次的时间成本（约 15~25s）必须**远小于**被卷入转盘的 90 秒 |
| 1 名程序可实现 | 用简单的"声源点调查 + 视线 + 计时器"，**不做复杂行为树** |

## 一、声音 = 玩家动作的信号半径

### 1.1 核心规则
> 玩家做出任何动作时，会在**该动作发生的位置**生成一个**声音半径**。如果饿魔此刻在这个半径之内，它就被"惊动"，并前往**声源点**（动作发生的位置，不是玩家当前位置）开始调查。

关键点：**饿魔追的是"声音发生的地方"，不是"玩家现在在哪"**。所以玩家发声后立刻安静走开，饿魔会扑向一个空位置并扑空——这正是"躲优先"成立的底层逻辑。

### 1.2 动作 → 声音半径表（v1，米，待调）

| 动作 | 声音半径 | 噪音级 |
|------|---------|--------|
| 静步 / 潜行 | 1.5m | 近乎无声 |
| 普通行走 | 5m | 低 |
| 奔跑 | 12m | 高 |
| 搜索 · 低噪（纸箱 / 货架） | 4m | 低 |
| 搜索 · 中噪（冰柜 / 保温柜） | 8m | 中 |
| 敲冰 · 碎裂瞬间 | 10m | 中偏高 |
| 钥匙 · 翻找（垃圾桶等） | 6m | 中 |
| 钥匙 · 开锁 | 3m | 低 |
| 凌波渡电 · 落水 / 电弧 | 15m | 高 |
| 投掷 / 丢道具 | 3m | 低 |

> 这张表与《场景互动小游戏全集》《玩家躲避系统》共用，改动需三处同步。

### 1.3 道具接口
| 道具 | 对声音系统的作用 |
|------|----------------|
| 静音胶带 | 下一次搜索 / 敲冰 / 开锁的声音半径 ×0.4（或封顶 4m）——可在饿魔附近安全开高价值容器 |
| 臭包 | 本身不发声；落点 **10 秒内成为饿魔"回避区"**，强行把巡逻/调查方向推离该区域 |

## 二、饿魔状态机（四态）

> 极简状态机，便于 1 程序实现与调试。

```text
        无新线索超时(~15~25s)
   ┌─────────────────────────────────────────┐
   │                                          │
   ▼                                          │
[巡逻 Patrol] ──声音半径内被惊动──▶ [调查 Investigate] ──持续/再受惊──▶ [搜寻 Search]
   ▲                                  │                                    │
   │                                  │ 近距离(<8m)清晰视线 ~1.5s           │ 同左
   │                                  ▼                                    ▼
   └──────────────────────────── [锁定 Lock] ◀───────────────────────────┘
                                       │
                                       ▼
                              3 秒吸入 → 转盘供宴（代价事件）
```

### 2.1 各状态定义

| 状态 | 行为 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| **巡逻 Patrol** | 沿 3 条预设路线之一移动（见《地图尺寸与路线规划》），不感知玩家 | 默认 / 放弃搜寻后回归 | 声音半径内被惊动；或近距离视线发现玩家 |
| **调查 Investigate** | 前往**声源点**，到达后在附近搜索 8~12s | 在声音半径内听到动作 | 期间无新线索→进入放弃倒计时；收到新声音/视线→升级为搜寻；近距离视线→锁定 |
| **搜寻 Search** | 在更大范围内更积极地搜索 12~15s，移动更快 | 调查期间再次被惊动或瞥见玩家 | 无新线索→放弃→回巡逻；持续近距离视线→锁定 |
| **锁定 Lock** | 锁定玩家，张口吸入 | 近距离(<8m)清晰视线持续 ~1.5s；或搜寻态多次极近线索 | 进入 3 秒吸入演出 → 转盘 |

### 2.2 视线（次要触发，让"断视线"有意义）
- 饿魔有一个**视野锥 + 视距**（v1：约 8~10m，锥角 90°）。
- 在视野锥内且无遮挡 → 视线命中，**快速推向锁定**。
- 墙体、建筑、容器、躲藏点都能**遮断视线**。
- 设计意图：声音负责"把饿魔引过来"，视线负责"最后确认/锁定"。玩家既要管声音，也要管"别让它直视到你"。

### 2.3 关键计时器（v1，让"躲"在时间上稳赢）

| 计时器 | v1 值 | 作用 |
|--------|-------|------|
| 调查搜索时长 | 8~12s | 到达声源点后搜索多久 |
| 搜寻搜索时长 | 12~15s | 升级后搜索多久 |
| 放弃倒计时 | 无新线索后 ~10s | 之后回归巡逻 |
| 一次惊动的总时间成本 | **≈ 15~25s** | 玩家"被惊动→安静/躲→饿魔放弃"的总耗时 |
| 视线锁定阈值 | 近距离清晰视线 ~1.5s | 触发锁定 |

> **核心数值结论**：被惊动一次 ≈ 15~25s ＜ 转盘 90s。所以**安静走开 / 躲一下**永远比"被抓进转盘"省时——这就是"躲优先"的数值保证。

### 2.4 时间难度递增（追人速度随单局时间提升 · 已拍板）

> 饕餮强度随单局时间推移而增强。上面的 v1 计时器为**基准（中期 T2）**值，按时间档缩放：

| 档位 | 单局时间 | 巡逻/搜寻移速 | 放弃倒计时 | 视线锁定阈值 | 体感 |
|------|---------|--------------|-----------|-------------|------|
| T1 温和 | 0–5min | 巡逻×0.85 / 搜寻基准 | 短 ~8s | ~1.8s | 容易甩开，降噪走开即可 |
| T2 升温 | 5–10min | 基准 / ×1.1 | ~10s | ~1.5s | 要认真断视线 / 钻躲点 |
| T3 刁钻 | 10–15min | ×1.15 / ×1.25 | 长 ~12s | ~1.2s | 很难空手甩开，常需臭包 / 引怪 / 躲点 |
| 狂暴/迷失线 | **15:00 硬上限** | 最快 | 最长 | 最短 | 最后冲刺；**未撤=迷失失全部背包** |

**护栏（守住"躲优先"）**：一次惊动总成本随档略升（T1 ~15s → T3 ~25~30s）但**始终 ＜ 转盘 90s**。递增靠**速度与锁定激进度**，不靠拉长时长——是"越后期越难成功躲掉"，而非"躲本身变得不划算"。被抓后进入的转盘也按同一时间档加难（见《可配置参数总表 v1》§13.4、《双环转盘供宴改版草案 v3》§0.2–0.3）。

## 三、合作维度（声音是共享的，饿魔只有一个）

> 这是声音系统最出彩的合作/喜剧来源。

- 饿魔只有一只，但**三名玩家各自发声**。一个人在 A 处奔跑，可能把饿魔从队友所在的 B 处吸引过去——也可能把它**引向**正在安静开锁的队友。
- 由此自然产生喊话：
  - "你别在那儿跑啊，它要过来了！"
  - "我帮你引开它，你赶紧开那个锁。"（主动制造噪音 / 丢臭包**引怪**）
  - "我这边要敲冰了，先确认下它在哪。"
- **主动引怪**是一种合作战术：一人用奔跑/投掷把饿魔拉离，另两人在安全窗口里完成高噪解障或搜刮。臭包则反向把饿魔从某区域推开。

## 四、与"终极目标"的关系

| 玩家诉求 | 声音系统如何制造取舍 |
|---------|---------------------|
| 想搜得多、搜得快 | 越快越吵（奔跑 12m、中噪容器 8m）→ 越容易招饿魔 → 越可能损失时间 |
| 想躲、想省时 | 安静走开/躲起来能让饿魔放弃（15~25s）→ 但躲着不动食物在变质（见《单局节奏设计》） |
| 想开高价值容器 / 解障 | 高噪动作（敲冰 10m、渡电 15m）风险大 → 用静音胶带、先确认饿魔位置、或引怪开窗口 |

声音系统把"贪 vs 稳"的核心取舍，从"时间"延伸到了"暴露风险"。

## 五、灰盒验证清单（这套系统该怎么测）

> 你们已有核心循环 demo，可在其上叠加本系统单独验证。每条只测一个问题：

1. 饿魔"扑向声源点而非玩家"是否清楚？玩家能否凭直觉"发声后走开就甩掉它"？
2. 一次惊动的时间成本是否明显小于 90s？玩家是否自然选择"躲"而不是"打"？
3. 声音半径是否可读？玩家是否能感知"我这个动作有多吵"（需要美术/UI 提示，见下）？
4. 三人时是否出现"引怪 / 别在那儿跑"这类喊话？
5. 视线遮断是否有用？玩家是否会主动用墙/容器挡视线？

### 可读性提示（给美术/UI）
- 玩家发声时，在声源处显示一圈**扩散的声波环**（半径=声音半径），让玩家"看见自己有多吵"。
- 饿魔进入调查/搜寻态时，给出明确状态提示（如头顶问号 / 感叹号 + 预警红光分级，沿用 v7.3 §8.1 的远/中/近脉冲）。
