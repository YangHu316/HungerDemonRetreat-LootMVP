## 一、玩法规则

1. **观察（前摇约 2s）**：一池通电水域横在路上（电弧噼啪、水面蓝白电光）。水面上有一串**落脚点**（锈管/水泥块/漂浮外卖箱），间距不等，构成跨越路线。
2. **首名蓄力跳跃**：玩家**长按蓄力**（蓄力条上涨→对应跳跃距离），松开起跳，沿**抛物线**落向下一落脚点。
   - 落点有**预测线/落点圈**辅助瞄准（可调精度）。
   - 蓄力不足/过头 → 落进水里 → **触电弹飞、弹回起点、损失时间**（不致死）。
3. **拉电闸（开路）**：首名玩家抵达对岸终点，按 **E** 拉下电闸 → 全池**永久断电** → 电光熄灭、水面恢复安全。
4. **后续轻松通过**：断电后，另两名玩家可直接**趟水走过**（或落脚点变安全可随意踩），每人仅 3~5s。

> **设计灵魂**：把"渡电"做成**一个人承担紧张、为全队打开通路**。首名玩家的精准跳跃压力，与后续玩家的轻松形成强反差。

## 二、流程与状态机

```text
[通电 Charged] ──首名观察(2s)──▶ [蓄力跳 ChargeJump] ──循环跳过每个落脚点──┐
     ▲                              │ 落水 → 触电弹回起点(损时) ↺            │
     │                              └──────────────────────────────────────┘
     │                                            │ 抵达对岸终点
     │                                            ▼
     │                                   [拉电闸 Switch(按E)]
     │                                            │
     ▼                                            ▼
（后续玩家在断电后）◀───────────── [断电 PoweredOff·永久安全] ──▶ 后续两人趟水通过(3~5s/人)
```

| 状态 | 行为 | 进入 | 退出 |
|------|------|------|------|
| Charged | 水池通电、电弧演出、落脚点可踩但落水危险 | 默认 | 首名开始跳 |
| ChargeJump | 蓄力→起跳→落点判定，逐个落脚点推进 | 首名起跳 | 落水(弹回) / 抵达终点 |
| Switch | 终点拉电闸交互 | 抵达对岸 | 拉下→断电 |
| PoweredOff | 全池断电、水面安全、**全局永久** | 拉闸成功 | — |
| （后续通过） | 其余玩家趟水走过 | 断电后 | 全队过池→完成 |

## 三、时长预期（详细）

| 阶段 | 首名玩家 | 后续玩家 |
|------|---------|---------|
| 前摇 | 观察路线 2 s | — |
| 核心 | 蓄力跳过 N 个落脚点 + 拉电闸 | 趟水安全走过 |
| 反馈 | 落水→电弧弹飞（重试）/ 成功→电光熄灭、水面恢复 | — |
| **总计** | **15~25 s**（含可能的 1~2 次落水重试） | **3~5 s** |

- 这是**程序量最大、最易出联机 bug** 的解障之一（与双环并列）。
- 落脚点数与间距决定首名时长与难度（v1：4~6 个落脚点）。

## 五、镜头设计（本作镜头难点）

> **核心问题**：纯俯视角看不清"跳起来的高度与落点"。渡电是唯一含**纵向跳跃**的解障，镜头必须特殊处理，否则跳跃不可玩。

| 阶段 | 镜头行为 |
|------|---------|
| 进入渡电 | 0.5s 从俯视 lerp 到 **~30~40° 侧后方斜俯视**，压低俯角，让**落脚点间距 + 跳跃弧线 + 危险水面**都看得清 |
| 蓄力 | 镜头略拉近 + 蓄力条聚焦角色；可轻微"屏息"感（FOV 微收） |
| 起跳/飞行 | 镜头**跟随抛物线**，强调腾空与落点 |
| 落水 | 小幅震屏 + 电弧闪白 + 角色弹飞，镜头快速回扫到起点（重试） |
| 拉闸成功 | 镜头**横扫整个断电水池**（电光逐段熄灭、水面恢复安全色），给"我把路打开了"的成就感，也让岸边队友看清"安全了" |
| 后续玩家 | **保持常规俯视**（无需斜视）——用镜头差异强化"轻松"反差 |
| 三人 | **非传送**：首名过池时队友在岸边世界空间观战、喊话；只有首名进斜俯视聚焦 |

**要点**：① 斜俯视角度要让落点距离可判断（这是蓄力依据）；② 拉闸后的"扫池"镜头是情绪付费点，别省；③ 首名/后续的镜头反差服务"开路"叙事——🔴 人调。

## 六、美术风格与资产清单

**风格基调**：美食废土 × 低模卡通；危险水面与安全水面、落脚点与水必须强对比可读。

| 资产 | 说明 | 俯视/斜俯视可读重点 | 优先级 |
|------|------|---------------------|--------|
| 通电水池 | 废墟积水 + 电弧噼啪 + 水面蓝白 electric shimmer | "这水带电、碰了完蛋"一眼可知 | 🔴 必做 |
| 落脚点 | 锈管/水泥块/漂浮外卖箱，间距不等 | 可踩点清晰、与水对比强 | 🔴 必做 |
| 终点电闸 | 旧式工业闸刀开关 | 醒目可交互、"拉这个"直觉 | 🔴 必做 |
| 断电演出 | 电光逐段熄灭、水面转安全暗色 | 安全/危险状态切换明确 | 🔴 必做 |
| 落水触电弹飞 | 电弧 + 角色弹飞 + 焦糊烟 | 滑稽不致死、爽快 | 🟡 |
| 蓄力/起跳/落地姿态 | 角色压低蓄力 → 腾空 → 落地 | （纸片角色尤其需要清晰姿态帧） | 🔴 必做（手感可读） |

## 七、声音接口（接《感知与寻人》系统）

| 事件 | 声音半径 | 噪音级 | 说明 |
|------|---------|--------|------|
| 蓄力跳·落地 | ~4~5 m | 低中 | 每跳落到落脚点的轻响（〔新增〕待调） |
| 落水 / 电弧 | **15 m** | 高 | **全场最大声的解障**；发声点=水池。深区/商业区深处饿魔活跃 → 渡电天然高压 |
| 拉电闸 | ~4 m | 低 | 终点拉闸的机械响（〔新增〕待调） |

> 与《感知与寻人》§1.2 / 《参数总表》§7 同步：渡电三类发声＝跳落地 ~4~5m、落水/电弧 15m、拉闸 ~4m。

- 这是噪音风险最高的解障：跳之前最好先确认饿魔位置，或队友引怪/丢臭包开窗口。
- 落水重试会**反复发出 15m 巨响**——多次失败=持续招饿魔，把"精准跳"的压力翻倍。
- 静音胶带对落水电弧**作用有限**（高噪事件）——设计上倾向"靠走位躲 + 引怪"而非压噪。

## 八、三人合作价值

- **一人冒险开路**：第一个过池拉闸的人承担全部紧张；成功后另两人轻松趟过。
- 岸边队友可**喊话指挥**（"那块石头太远，先跳近的""它来了快跳！"）、**引怪掩护**（一人把饿魔引开，让首名安心跳）。
- 反差体验是灵魂：**首名的手心冒汗** vs **后续的闲庭信步**。

## 十、网络同步要点（局域网三人）⚠️ 高风险

| 项 | 处理 |
|----|------|
| 权威 | **主机权威**：跳跃位置/弹道、落水判定、电闸状态、弹回均由主机裁决 |
| 客户端 | 首名跳跃用客户端预测 + 插值，主机校正落点/落水 |
| 电闸状态 | **必须强一致**：断电是全局永久状态，三人画面必须同时切到"安全"，否则有人趟水触电、有人没——体验崩坏 |
| 弹回 | 落水弹回起点要同步，避免位置漂移 |
| 风险 | **高**——蓄力跳的位置同步 + 电闸全局一致是隐蔽 bug 高发区。**与双环并列，必须程序亲测 3 真机，不可全交 AI**（再平衡 v2 §3.1 / 执行案 §四铁律） |

## 十二、灰盒验证清单

1. **首名玩家的紧张 vs 后续玩家的轻松，反差成立吗**？（灵魂指标）
2. **蓄力跳的手感舒服吗**？落点预测够不够直觉、跳跃曲线爽不爽？
3. 斜俯视镜头能让玩家判断落点距离吗？
4. 落水弹回 + 15m 巨响是否制造"别失误"的真实压力，而不是纯挫败？
5. 三人时电闸断电后，三台机器是否**同时**看到水面安全（联机一致性）？

## 十三、白盒最低形态（可先于美术）

通电水池区域 + 4~6 个可跳落脚点 + 蓄力跳跃（蓄力条 + 落点预测）+ 终点电闸 + 断电后"安全水面"状态切换 + 落水弹飞占位。**不需要**：电弧特效、水池 shader、电闸动画、最终音效。**但联机三人一致性必须在白盒阶段就测**（不能等美术）。
